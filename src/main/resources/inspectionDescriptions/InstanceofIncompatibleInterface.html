Сообщает о выражениях instanceof, где проверяемый тип класса/интерфейса не расширяет и не реализует тип класса/интерфейса справа от instanceof, а также не имеет подклассов, которые это делают.

Хотя такая конструкция может использоваться в библиотечном коде, она, вероятно, ошибочна, поскольку ни один экземпляр любого класса, объявленного в проекте, не сможет пройти эту проверку instanceof.

Пример:


  class Foo { }
  interface Bar { }
  class Main {
      void test(Foo f, Bar b) {
          if (f instanceof Bar) { // Проблема
              System.out.println("fail");
          }
          if (b instanceof Foo) { // Проблема
              System.out.println("fail");
          }
      }
  }
