Обнаружены вызовы логования, не защищенные условными операторами. Вычисление аргументов для таких вызовов может быть ресурсоемким. Оборачивание вызовов логования в условные блоки позволяет избежать этих затрат, когда логование для данного уровня отключено. Это особенно полезно для наименее критичных уровней логования (trace, debug, finest), поскольку они чаще всего отключены в производственной среде.

Пример:


  public class TestObject {
    void test(Object object) {
      LOG.debug("some logging " + expensiveCalculation(1));
    }
  }

После применения быстрого исправления:


  public class TestObject {
    void test(Object object) {
      if(LOG.isDebugEnabled()){
        LOG.debug("some logging " + expensiveCalculation(1));
      }
    }
  }

Эта проверка поддерживает фреймворки логования Log4j2 и SLF4J (за исключением построителей).
