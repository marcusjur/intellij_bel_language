Начиная с Kotlin 1.4, сообщается об избыточных метках, вызывающих ошибки компиляции.

Начиная с Kotlin 1.0, можно было помечать метками любые выражения:


  fun foo() {
    L1@ val x = L2@bar()
  }
Однако эти метки можно использовать только несколькими ограниченными способами:
  • прерывание/продолжение цикла
  • нелокальный возврат из встроенной лямбды или встроенной анонимной функции

Начиная с Kotlin 1.4, использование таких меток запрещено.

Эта проверка выполняется только если уровень языка Kotlin для проекта или модуля установлен на 1.4 или выше.
