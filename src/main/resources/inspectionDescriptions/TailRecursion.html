Сообщает о хвостовой рекурсии, где метод вызывает сам себя в качестве последней операции перед возвратом.

Хвостовая рекурсия всегда может быть заменена циклом, который работает значительно быстрее. Некоторые JVM выполняют оптимизацию хвостовых вызовов, а другие - нет. Следовательно, решения с хвостовой рекурсией могут иметь кардинально разные характеристики производительности на разных виртуальных машинах.

Пример:


  int factorial(int val, int runningVal) {
    if (val == 1) {
      return runningVal;
    } else {
      return factorial(val - 1, runningVal * val);
    }
  }

После применения быстрого исправления:


  int factorial(int val, int runningVal) {
    while (true) {
      if (val == 1) {
        return runningVal;
      } else {
        runningVal = runningVal * val;
        val = val - 1;
      }
    }
  }
