Сообщает об аннотации @GuardedBy, в которой защищаемое поле не является final.

Защита нефинального поля может привести к непредвиденным условиям гонки, поскольку блокируется значение поля (которое может измениться), а не само поле.

Пример:


    private ReadWriteLock lock = new ReentrantReadWriteLock(); //нефинальное защищаемое поле
    private Object state;
    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Поддерживаемые аннотации @GuardedBy:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
