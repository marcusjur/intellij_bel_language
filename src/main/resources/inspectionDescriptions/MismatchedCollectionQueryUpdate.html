Сообщает о коллекциях, содержимое которых было запрошено, но не обновлено, или обновлено, но не запрошено.

Такие несоответствия между запросами и обновлениями бессмысленны и могут указывать на наличие мертвого кода или опечаток.

В настройках проверки можно указать шаблоны имен, соответствующие методам обновления/запроса. Методы запроса, возвращающие элементы, определяются автоматически, и нужно указывать только методы запроса, которые записывают данные в выходные параметры (например, OutputStream).

Пример:

Предположим, у вас есть пользовательский класс FixedStack с методом store(): 


  public class FixedStack<T> extends Collection<T> {
    public T store(T t) {
      // реализация
    }
  }

Вы можете добавить store в таблицу методов обновления, чтобы получать сообщения о несоответствующих запросах, например: 


  void test(int i) {
    FixedStack<Integer> stack = new FixedStack<>();
    stack.store(i);
  }
