Обнаружение JSON-файлов, которые можно интерпретировать как часть спецификации OpenAPI/Swagger.

Эти файлы не содержат явных признаков спецификации, но они ссылаются на основной файл спецификации, находящийся поблизости.

В примере ниже показаны два файла, находящиеся в одном каталоге. Первый — это обычный основной файл спецификации. Второй ссылается на первый, поэтому рекомендуется рассматривать его как спецификацию.

Основной файл спецификации openapi.yaml:


openapi: 3.1.0
components:
  schemas:
    CustomSchema:
      description: Custom schema object
      properties:
        foo:
          $ref: 'common.components.json#/components/schemas/CommonSchema'

Предполагаемый файл спецификации common.components.json:


{
  "components": { // 'Mark file as OpenAPI specification' highlighting
    "schemas": {
      "CommonSchema": {
        "description": "Common schema object reused in several specifications"
      }
    }
  }
}
