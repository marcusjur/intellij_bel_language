Обнаружен служебный класс, не помеченный как final или abstract.

Служебный класс объявляет все поля и методы как static. Объявление его как final предотвращает случайное наследование.

Пример:


  public class UtilityClass {
    public static void foo() {}
  }

После применения быстрого исправления:


  public final class UtilityClass {
    public static void foo() {}
  }
