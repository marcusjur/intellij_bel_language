Сообщает о проверках на null и предлагает заменить их цепочками Optional.

Пример:


  return str == null ? "" : str.trim();

После применения быстрого исправления:


  return Optional.ofNullable(str).map(String::trim).orElse("");

Хотя замена не всегда приводит к сокращению кода, она может способствовать дальнейшему рефакторингу (например, изменению возвращаемого значения метода на Optional).

Обратите внимание, что когда ветвь условия для не-null значения возвращает null, соответствующий шаг отображения создаст пустой Optional, что может изменить семантику. Если невозможно статически доказать сохранение семантики, название быстрого исправления будет содержать уведомление "(возможно изменение семантики)", и подсветка проверки будет отключена.
