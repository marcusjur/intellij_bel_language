Обнаружены следующие проблемы в XML-файле конфигурации:
  • Ссылки на классы, которые невозможно инстанцировать
  • Ссылки на классы, которые не расширяют требуемые классы
  • Ссылки на классы с неправильной областью видимости
  • Теги и значения атрибутов, не соответствующие требуемому шаблону (например, Java-идентификаторы)
  • Теги без обязательных дочерних тегов или атрибутов
  • Теги, определяющие объекты с повторяющимися именами

Пример:



<persistence>
    <persistence-unit name="example-persistence-unit">
        <provider>org.example.SomeRandomClass</provider> // Ошибка: класс провайдера должен реализовывать интерфейс PersistenceProvider
    </persistence-unit>
</persistence>
