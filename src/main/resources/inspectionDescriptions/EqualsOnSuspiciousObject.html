Сообщает о вызовах метода equals() для экземпляров StringBuilder, StringBuffer и классов из пакета java.util.concurrent.atomic.

Метод equals() не переопределен в этих классах, поэтому может вернуть false, даже если содержимое двух объектов идентично. Если вы намереваетесь проверить равенство ссылок, рекомендуется использовать == во избежание путаницы.

Доступно быстрое исправление для StringBuilder, StringBuffer, AtomicBoolean, AtomicInteger, AtomicBoolean и AtomicLong для преобразования в сравнение содержимого. Быстрое исправление может изменить семантику, когда один из экземпляров равен null.

Пример:


  public void test(StringBuilder sb1, StringBuilder sb2) {
    boolean result = sb1.equals(sb2); // подозрительно
  }

После применения быстрого исправления:


  public void test(StringBuilder sb1, StringBuilder sb2) {
      boolean result = sb1.toString().equals(sb2.toString());
  }
