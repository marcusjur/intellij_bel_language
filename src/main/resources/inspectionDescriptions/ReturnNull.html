Сообщает об операторах return, возвращающих значение null. Хотя иногда это может быть полезно, такая структура может привести к сбоям в коде из-за повышенной вероятности возникновения NullPointerException.

Если метод спроектирован для возврата null, рекомендуется пометить его аннотацией @Nullable - такие методы будут игнорироваться данной проверкой.

Пример:


  class Person {
    public String getName () {
      return null;
    }
  }

После применения быстрого исправления:


  class Person {
    public @Nullable String getName () {
      return null;
    }
  }

Если возвращаемый тип - java.util.Optional, рекомендуется использовать другое быстрое исправление для преобразования null в Optional.empty().

Настройте проверку, используя следующие параметры:

  • Следует ли игнорировать private методы. Это также будет игнорировать возврат null из анонимных классов и лямбда-выражений.
  • Следует ли сообщать о возврате null для массивов, коллекций, обычных объектов или их комбинации. Всегда сообщается о возврате null в методах, возвращающих тип java.util.Optional.
  • Нажмите Настроить аннотации, чтобы указать, какие аннотации следует рассматривать как 'nullable'.
