Сообщает о неиспользуемых или недоступных из точек входа классах, методах или полях.

Точками входа могут быть метод main, тесты, классы вне указанной области, классы, доступные из module-info.java и т.д. Пользовательские точки входа можно настроить с помощью шаблонов имен или аннотаций.

Пример:


  public class Department {
    private Organization myOrganization;
  }

В этом примере Department явно ссылается на Organization, но если класс Department сам не используется, проверка отметит оба класса.

Проверка также сообщает о неиспользуемых параметрах, методы, реализации и переопределения которых не используются, а также об объявленных, но неиспользуемых локальных переменных.

Примечание: Некоторые неиспользуемые элементы могут не отмечаться при подсветке кода в редакторе. По соображениям производительности проверка выполняется только для non-private элементов, имена которых редко встречаются в проекте. Чтобы увидеть все результаты, запустите проверку, выбрав Код | Проверить код или Код | Анализовать код | Запустить проверку по имени в главном меню.

Используя настройки видимости ниже, можно настроить, о каких элементах следует сообщать. Например, настройка отчётности для private методов означает, что будут отмечаться public методы во внутренних private классах, а protected методы в классах верхнего уровня будут игнорироваться.

Используйте вкладку Точки входа, чтобы настроить точки входа, которые следует учитывать при выполнении проверки.

После подготовки результатов проверки точки входа можно добавить вручную.

Если ваш код использует неподдерживаемый фреймворк, есть несколько вариантов:

  • Если фреймворк основан на аннотациях, используйте кнопку Аннотации... для настройки аннотаций фреймворка.
  • Если фреймворк не использует аннотации, попробуйте настроить шаблоны имен классов, которые должны быть во фреймворке.

Таким образом, код с аннотациями, доступный внутри фреймворка, будет считаться использованным.
