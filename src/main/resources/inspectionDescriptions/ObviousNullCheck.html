Сообщает о вызовах методов проверки на null (таких как Objects.requireNonNull или Assert.assertNotNull) для значений, которые очевидно не могут быть null (например, для только что созданных объектов). Такие проверки избыточны и могут указывать на ошибку в программовании.

Пример:


  final String greeting = Objects.requireNonNull("Hi!");

После применения быстрого исправления:


  final String greeting = "Hi!";

Изменения в версии 2017.2
