Сообщает о полях, которые можно безопасно объявить как final. Все поля final должны иметь неизменяемое значение, что делает код более понятным для анализа.

Во избежание чрезмерных накладных расходов на анализ, эта проверка сообщает только о полях, которые имеют модификатор private или определены в локальном или анонимном классе. Поле может быть объявлено как final, если:

  • Поле является static и инициализуется один раз при объявлении или в static инициализаторе.
  • Поле не является static и инициализуется один раз при объявлении, в инициализаторе экземпляра или в каждом конструкторе
и поле не модифицируется в каком-либо другом месте.

Пример:


  public class Person {
    private String name; // может быть final
    Person(String name) {
      this.name = name;
    }
    public String getName() {
      return name;
    }
  }

После применения быстрого исправления:


  public class Person {
    private final String name;
    Person(String name) {
      this.name = name;
    }
    public String getName() {
      return name;
    }
  }

Используйте кнопку "Аннотации" для изменения списка аннотаций, предполагающих неявную запись в поле.
