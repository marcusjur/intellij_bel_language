Сообщает об использовании устаревших классов, полей и методов. Когда необходимая информация может быть извлечена из Javadoc устаревшего элемента, быстрое исправление может автоматически преобразовать устаревшее использование.

Пример:


  class Interesting {
    /**
     * @deprecated Используйте {@link #newHotness()}
     */
    @Deprecated
    public void oldAndBusted() {}
    public void newHotness() {}
  }
  class ElseWhere {
    void x(Interesting i) {
      i.oldAndBusted(); // Здесь будет предупреждение об устаревшем методе
    }
  }

После применения быстрого исправления:


  class Interesting {
    /**
     * @deprecated Используйте {@link #newHotness()}
     */
    @Deprecated
    public void oldAndBusted() {}
    public void newHotness() {}
  }
  class ElseWhere {
    void x(Interesting i) {
      i.newHotness();
    }
  }

По умолчанию проверка не выдает предупреждения, если использования устаревшего кода невозможно или сложно избежать. Например, следующий код не будет отмечен:


  abstract class A { //код библиотеки
     @Deprecated
     abstract void m();
  }
  class B extends A { //код проекта
     @Override
     void m() {
        //doSmth;
     }
  }

Настройка проверки:

С помощью опций можно отключить эту проверку для устаревших элементов, переопределений абстрактных устаревших методов, операторов non-static import, методов устаревших классов или в том же классе верхнего уровня.
