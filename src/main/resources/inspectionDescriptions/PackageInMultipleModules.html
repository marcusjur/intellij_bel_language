Обнаружены непустые пакеты, существующие в нескольких модулях. Когда пакет включен в несколько модулей, легко создать классы с одинаковыми именами в обоих модулях. Конфликт возникает, когда модуль, зависящий от этих модулей, пытается использовать такой класс. Java Platform Module System не допускает пакеты, содержащиеся в нескольких модулях (также известные как split-пакеты)
