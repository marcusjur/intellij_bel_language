Сообщает о вызовах фабричных методов MethodHandle и VarHandle, которые не соответствуют никаким методам или полям.

Также сообщает о несоответствии аргументов для MethodHandle.invoke() и подобных методов с сигнатурой MethodHandle, а также о несоответствии аргументов для VarHandle.set() с типом VarHandle

Пример:

  MethodHandle mh = MethodHandles.lookup().findVirtual(
      MyClass.class, "foo", MethodType.methodType(void.class, int.class));
  // Этот аргумент должен быть значением типа int
  mh.invoke(myObj, "abc");


  // Этот аргумент должен быть String.class
  VarHandle vh = MethodHandles.lookup().findVarHandle(
      MyClass.class, "text", int.class);


  VarHandle vh = MethodHandles.lookup().findVarHandle(
      MyClass.class, "text", String.class);
  // Этот аргумент должен быть значением типа String
  vh.set(myObj, 42);
