Сообщение о конфликтующих атрибутах в HCL-блоке.

Пример проблемы:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
  version = "~> 1.24"
}

В этом примере атрибут version объявлен дважды в блоке провайдера Kubernetes. Это является ошибкой.

Пример исправления:


provider "kubernetes" {
  version = "~> 1.25"
  config_path = "~/.kube/config"
}
Техническая поддержка: Плагин Terraform и HCL
