Обнаружен класс-одиночка (Singleton).

Класс-одиночка объявляется таким образом, что может быть создан только один экземпляр этого класса. Использование паттерна Singleton усложняет тестование и может указывать на недостатки в объектно-ориентированном проектировании.

Пример:


  class Singleton {
    private static final Singleton ourInstance = new Singleton();
    private Singleton() {
    }
    public Singleton getInstance() {
        return ourInstance;
    }
  }
