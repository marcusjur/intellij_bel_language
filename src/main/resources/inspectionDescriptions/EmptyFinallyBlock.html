Обнаружение пустых блоков finally.

Пустые блоки finally обычно указывают на ошибки в коде. Они могут остаться после рефакторинга кода и могут быть безопасно удалены.

Эта проверка не распространяется на пустые блоки finally, обнаруженные в JSP файлах.

Пример:


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  } finally {
  }

После применения быстрого исправления: 


  try {
    Files.readString(Paths.get("in.txt"));
  } catch (IOException e) {
    throw new RuntimeException(e);
  }
