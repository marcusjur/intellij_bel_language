Сообщает об именах классов, которые не соответствуют рекомендуемым соглашениям об именовании.

Последовательное именование упрощает чтение и понимание кода. Согласно официальному руководству по стилю Kotlin, имена классов должны начинаться с заглавной буквы и использовать CamelCase.

Дополнительные правила именования можно ввести, изменив регулярное выражение "Pattern".

Пример:


  class user(val name: String)

Быстрое исправление переименует класс в соответствии с соглашениями Kotlin:


  class User(val name: String)
