Строковые шаблоны используются как аргументы для методов логования SLF4J и Log4j 2. Метод org.apache.logging.log4j.Logger.log() и его перегрузки поддерживаются только для опции все уровни логования. Строковые шаблоны вычисляются во время выполнения даже если сообщение не логируется; это может негативно влиять на производительность. Рекомендуется использовать параметризованные сообщения логования, которые не вычисляются, когда логование отключено.

Пример (для Kotlin):


      val variable1 = getVariable()
      logger.info("variable1: $variable1")

После применения быстрого исправления (для Kotlin):


      val variable1 = getVariable()
      logger.info("variable1: {}", variable1)

Обратите внимание, что предлагаемая замена может не быть эквивалентной исходному коду, например когда строковый шаблон содержит вызовы методов или выражения присваивания.

  • Используйте список игнорируемые уровни для исключения определённых более высоких уровней логования. Более высокие уровни логования могут быть всегда включены, и аргументы будут всегда вычисляться.
  • Используйте опцию не выдавать предупреждения, если содержатся только выражения с примитивными типами, их обёртками или строками для игнорирования строковых шаблонов, содержащих только выражения с примитивными типами, их обёртками или строками. Например, это может помочь предотвратить загрузку отложенных коллекций. Обратите внимание, что создание строк во время выполнения негативно влияет на производительность, даже если используются только выражения с примитивными типами, их обёртками или строками.
  • Используйте опцию не выдавать предупреждения для вызовов, где исключение передается как аргумент только после аргумента сообщения для игнорирования вызовов, где исключение передается как аргумент только после аргумента сообщения. Это может быть полезно, например, при настройке поведения проверки с помощью пользовательского стиля кода.
