Сообщает о параметрах, которые имеют названия, отличающиеся от соответствующих параметров в переопределённых или перегруженных методах. Хотя это допустимо в Java, такие несоответствия в именовании могут вызывать путаницу и снижают преимущества документации с правильными практиками именования.

Пример:


  class Person {
    Person(String fullName) {}
  }
  class Child extends Person {
    Child(String name) { super(name); }
  }

После применения быстрого исправления:


  class Person {
    Person(String fullName) {}
  }
  class Child extends Person {
    Child(String fullName) { super(fullName); }
  }

Параметры позволяют указать, следует ли игнорировать имена параметров длиной в один символ или параметры из методов библиотек при переопределении. Оба варианта полезны, если вы не хотите быть ограничены сомнительными соглашениями об именовании, используемыми в библиотеках.
