Аргумент вызова типа Boolean без явного указания имени параметра.

При последовательной передаче нескольких логических литералов легко забыть порядок параметров, что может привести к ошибкам. Явное указание имен параметров делает код более понятным и читаемым.

Пример:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}
  fun usage() {
      check(true, false, true) //что это значит?
  }

Быстрое исправление добавит отсутствующие имена параметров:


  fun check(checkName: Boolean, checkAddress: Boolean, checkPhone: Boolean) {}
  fun usage() {
      check(checkName = true, checkAddress = false, checkPhone = true)
  }
