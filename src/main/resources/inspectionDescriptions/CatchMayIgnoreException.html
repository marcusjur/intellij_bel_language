Сообщает о пустых блоках catch или блоках, которые могут игнорировать исключения.

Хотя иногда это делается намеренно, пустые блоки catch могут усложнить отладку. Кроме того, игнорирование параметров catch может быть ошибкой. Наконец, статический анализатор кода сообщит, если обнаружит, что блок catch может молча игнорировать важные исключения VM, такие как NullPointerException. Игнорирование таких исключений (без логования или повторного выброса) может скрыть ошибки.

Эта проверка не выдаёт предупреждения для параметров catch, названных ignore или ignored. Напротив, она предупреждает о параметрах catch с именами ignore или ignored, которые фактически используются. Кроме того, проверка не выдаёт предупреждения для параметров catch с именами expected или ok в тестовых исходниках.

Вы можете использовать быстрое исправление, чтобы переименовать исключение в ignored. Для пустых блоков catch предлагаются другие быстрые исправления для генерации тела блока catch. Вы можете изменить шаблон "Catch Statement Body" на вкладке "Код" в Настройки | Редактор | Файлы и шаблоны кода.

Пример:


  try {
    throwingMethod();
  } catch (IOException ex) {
  }

После применения быстрого исправления:


  try {
    System.out.println(System.in.read());
  } catch (IOException ignored) {
  }

Настройка проверки:

  • Используйте опцию Не выдавать предупреждения, когда блок 'catch' содержит комментарии, чтобы игнорировать блоки catch с комментариями.
  • Используйте опцию Не выдавать предупреждения, когда блок 'catch' не пустой, чтобы игнорировать блоки catch, содержащие операторы или комментарии, но не использующие саму переменную.
  • Используйте опцию Не выдавать предупреждения, когда исключение с именем 'ignore(d)' фактически используется, чтобы игнорировать используемые переменные с именем ignored.
