Обнаружены вызовы логования SLF4J или Log4j2 в классе, например logger.info("message: {}", key) с похожими сообщениями. Такие вызовы могут быть трудноразличимы, что затрудняет определение источника конкретного сообщения в логах.

Пример (Java):


  private static void request1(String text) {
      log.info("Message: {}", text); //похожий вызов
      doSomething1();
  }
  private static void request2(int i) {
      log.info("Message: {}", i); //похожий вызов
      doSomething2();
  }
  • Используйте опцию минимальная длина похожих последовательностей, чтобы установить минимальную длину последовательности, после которой будут отображаться предупреждения
  • Используйте опцию не сообщать о вызовах с уровнем логования 'error', чтобы игнорировать сообщения с уровнем `error`, содержащие исключения. Скрытие таких предупреждений может быть полезным, так как место вызова всё ещё можно определить по стек-трейсу
