Сообщает о классах, которые могут быть сериализованы или десериализованы.

Класс может быть сериализован, если он реализует интерфейс Serializable и его методы readObject() и writeObject() не определены так, чтобы всегда выбрасывать исключения. В коде, где важна безопасность, сериализуемые классы могут представлять опасность.

Пример:


  class DeserializableClass implements Serializable { // Этот класс не содержит метод 'writeObject()', выбрасывающий исключение
    private int sensitive = 736326;
    private void readObject(ObjectInputStream in) {
        throw new Error();
    }
}

После быстрого исправления:


  class DeserializableClass implements Serializable {
    private int sensitive = 736326;
    private void readObject(ObjectInputStream in) {
        throw new Error();
    }
    private void writeObject(java.io.ObjectOutputStream out) throws java.io.IOException {
        throw new java.io.NotSerializableException("DeserializableClass");
    }
  }

Настройте проверку, используя следующие параметры:

  • Укажите классы, наследники которых не должны отмечаться этой проверкой. Это применимо к классам, которые наследуют Serializable от суперкласса, но не используются для сериализации. Обратите внимание, что вместо игнорирования таких классов может быть безопаснее добавить методы readObject() и writeObject(), которые всегда выбрасывают исключения.
  • Следует ли игнорировать сериализуемые анонимные классы.
