Сообщает о циклах for, которые перебирают коллекции или массивы и могут быть автоматически заменены на улучшенный цикл for (синтаксис foreach).

Пример:


  for (Iterator<String> iterator = list.iterator(); iterator.hasNext(); ) {
    String item = iterator.next();
    System.out.println(item);
  }

После применения быстрого исправления:


  for (String item : list) {
    System.out.println(item);
  }

Опция сообщать о циклах по 'java.util.List' с использованием индекса позволяет находить циклы с вызовами list.get(index). Такие циклы обычно можно заменить на улучшенный цикл for, если только они не модифицируют список в процессе выполнения, например через вызов list.remove(index). В последнем случае улучшенный цикл for может вызвать исключение ConcurrentModificationException. В некоторых случаях циклы с list.get(index) также могут работать быстрее.

Опция не сообщать об итерациях по нетипизованным коллекциям позволяет игнорировать коллекции без параметров типа. Это предотвратит создание переменных улучшенного цикла for типа java.lang.Object и вставку приведения типов в местах использования переменной цикла.
