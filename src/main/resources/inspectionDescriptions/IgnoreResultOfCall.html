Сообщает о вызовах методов, результаты которых игнорируются.

Игнорирование результатов является абсолютно законным для многих методов, но для некоторых это почти наверняка ошибка. Примеры методов, для которых игнорирование результата может быть ошибкой, включают java.io.inputStream.read(), который возвращает фактическое количество прочитанных байтов, а также любые методы в классах java.lang.String или java.math.BigInteger. Эти методы не производят побочных эффектов, поэтому игнорирование их результатов бессмысленно.

Будут проверяться вызовы следующих методов:

  • Простые геттеры (методы, которые только возвращают поле)
  • Методы, указанные в настройках этой проверки
  • Методы, аннотированные с помощью org.jetbrains.annotations.Contract(pure=true)
  • Методы, аннотированные с помощью .*.CheckReturnValue
  • Методы в классах или пакетах, аннотированных с помощью javax.annotation.CheckReturnValue
  • Все небиблиотечные методы (опционально)

Не будут отмечаться вызовы методов, аннотированных с помощью @CanIgnoreReturnValue из Error Prone или AssertJ.

Используйте настройки проверки для указания классов, которые нужно проверять. Методы сопоставляются по имени или шаблону имени с использованием синтаксиса регулярных выражений Java. Для классов используйте полностью определённые имена. Каждая запись применяется к указанному классу и всем его наследникам.
