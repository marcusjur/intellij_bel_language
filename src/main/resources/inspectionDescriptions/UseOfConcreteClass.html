Сообщает о случаях использования конкретных классов вместо интерфейсов. Такие объявления могут указывать на недостаточную абстракцию и затруднять тестование.

Проверка не распространяется на объявления, классы которых происходят из системных или сторонних библиотек. В реализации метода equals() не отмечаются приведения типов, instanceof и локальные переменные. Кроме того, в реализации метода clone() не отмечаются приведения типов.

Пример:


  interface Entity {}
  class EntityImpl implements Entity {}
  void processObject(Object obj) {
    // Предупреждение: instanceof с конкретным классом
    if (obj instanceof EntityImpl) {
      // Предупреждение: приведение к конкретному классу
      // вместо интерфейса
      processEntity((EntityImpl)obj);
    }
  }
  // Предупреждение: параметр конкретного класса
  void processEntity(EntityImpl obj) {
  }

Используйте опцию Игнорировать типы абстрактных классов, чтобы игнорировать приведения к абстрактным классам.
