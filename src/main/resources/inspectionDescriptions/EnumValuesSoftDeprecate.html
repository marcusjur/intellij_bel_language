Сообщает о вызовах метода values() в перечислениях Kotlin, которые могут быть заменены на свойство entries.

Использование Enum.entries может улучшить производительность кода.

Быстрое исправление заменит values() на entries.

Подробнее: KT-48872 Современная и эффективная альтернатива для Enum.values()

Примечание: тип свойства entries отличается от возвращаемого типа метода values() (EnumEntries<T> наследуется от List<T>, а не от Array<T>). Поэтому в некоторых случаях быстрое исправление добавит дополнительное преобразование .toTypedArray(), чтобы не нарушить работу кода, но в большинстве случаев замена будет выполнена без этого преобразования (например, в циклах for).

Пример:


    enum class Version {
        V1, V2
    }
    Version.values().forEach { /* .. */ }
    val firstVersion = Version.values()[0]
    functionExpectingArray(Version.values())

После применения быстрого исправления:


    enum class Version {
        V1, V2
    }
    Version.entries.forEach { /* .. */ }
    val firstVersion = Version.entries[0]
    functionExpectingArray(Version.entries.toTypedArray())
