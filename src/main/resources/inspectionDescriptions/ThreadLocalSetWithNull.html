Обнаружен вызов java.lang.ThreadLocal.set() с null в качестве аргумента.

Этот вызов не освобождает ресурсы и может привести к утечке памяти.

Это происходит потому что:
  • Во-первых, ThreadLocal.set(null) ищет отображение, связанное с текущим потоком. Если такого отображения нет, оно создаётся
  • Затем устанавливаются ключ и значение: map.set(this, value), где this является экземпляром ThreadLocal

Для освобождения ресурсов следует использовать java.lang.ThreadLocal.remove().

Пример:


  ThreadLocal<Integer> threadLocal = new ThreadLocal<>();
  threadLocal.set(null);

После применения быстрого исправления:


  threadLocal.remove();
