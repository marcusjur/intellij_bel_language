Обнаружена конкатенация строк String в цикле.

Поскольку каждая конкатенация String создает копию всей строки, рекомендуется заменить её явным вызовом StringBuilder.append() или StringBuffer.append().

Пример:


  String str = "";
  for(int i=0; i<10; i++) {
    str += i;
  }

После применения быстрого исправления:


  String str = "";
  StringBuilder strBuilder = new StringBuilder(str);
  for(int i = 0; i<10; i++) {
    strBuilder.append(i);
  }
  str = strBuilder.toString();

В некоторых случаях это быстрое исправление может преобразовать переменную типа String в StringBuilder или добавить новый StringBuilder. Следует проявлять осторожность, если исходный код специально обрабатывает значения null, так как замена может изменить семантику. При возможности значений null рекомендуется применять null-безопасное исправление, которое создаст необходимые проверки на null. Кроме того, нет гарантии, что автоматическая замена всегда приведет к улучшению производительности.
