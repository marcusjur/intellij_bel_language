Метод createOptionsPanel() в реализации инспекции может быть автоматически преобразован в getOptionsPane().

Создание панели настроек инспекции через createOptionsPanel() считается устаревшим, вместо этого следует использовать декларативное описание элементов управления getOptionsPane(). Эта проверка анализует реализации createOptionsPanel() и автоматически предлагает переход на новый API, если они используют InspectionOptionsPanel или его наследников и достаточно просты.

В настоящее время проверка поддерживается только для Java и Kotlin. Она не может преобразовывать код, использующий методы Kotlin, такие как apply или also. Попробуйте выполнить их встраивание.
