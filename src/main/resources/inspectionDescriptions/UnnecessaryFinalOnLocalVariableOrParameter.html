Сообщает о локальных переменных или параметрах, которые необязательно объявлены как final.

Некоторые стандарты кодования не рекомендуют объявлять переменные как final из соображений лаконичности.

Пример:


  class Foo {
    Foo(Object o) {}
    void bar(final Object o) {
      new Foo(o);
    }
  }

После применения быстрого исправления:


  class Foo {
    Foo(Object o) {}
    void bar(Object o) {
      new Foo(o);
    }
  }

С помощью параметров проверки можно включить или отключить отчёты для:

  • Локальных переменных
  • Параметров (включая параметры блока catch и расширенного оператора for)

Кроме того, можно настроить проверку так, чтобы она сообщала только о параметрах final в abstract методах или методах интерфейса, которые могут считаться избыточными, поскольку такие модификаторы не влияют на реализацию этих методов.
