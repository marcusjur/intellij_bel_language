(Gradle 4.9+) Обнаружение использования API для предварительного взаимодействия с задачами.

Предварительное взаимодействие с задачами создает ряд проблем:

  • пользователям необходимо вручную устанавливать все зависимости между задачами;
  • на этапе конфигурации все задачи, к которым обращаются через предварительный API, будут сконфигурированы, даже если они не будут выполняться позже. Это приводит к снижению производительности.

В конечном итоге предварительный API будет объявлен устаревшим, и ему на смену придет отложенный API.

Руководство по миграции см. в документации Gradle.

Пример:

task foo { // сообщение о 'task', рекомендуется заменить на 'task.register'
  // ...
}
