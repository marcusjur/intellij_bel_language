Сообщает об операциях присваивания вида x = atomic.AddUint64(&x, 1).

Такие операции являются неатомарными, что представляет собой распространённую ошибку при использовании API sync/atomic. Чтобы сделать операцию атомарной, уберите присваивание и используйте прямой вызов: atomic.AddUint64(&x, 1). В этом случае значение x будет обновлено атомарно по адресу.

Пример:

import (
 "sync/atomic"
)
type Counter uint64
func AtomicTests() {
 x := uint64(1)
 x = atomic.AddUint64(&x, 1)         // Ошибка "прямое присваивание атомарному значению"
 _, x = 10, atomic.AddUint64(&x, 1)  // Ошибка "прямое присваивание атомарному значению"
 x, _ = atomic.AddUint64(&x, 1), 10  // Ошибка "прямое присваивание атомарному значению"
}
