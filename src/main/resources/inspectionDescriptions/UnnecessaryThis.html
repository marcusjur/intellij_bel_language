Сообщать о необязательных квалификаторах this.

Использование this для устранения неоднозначности ссылок в коде не рекомендуется во многих стилях программования и часто становится излишним благодаря автоматическому рефакторингу.

Пример:


  class Foo {
    int x;
    void foo() {
      this.x = 2;
    }
  }

После применения быстрого исправления:


  class Foo {
    int x;
    void foo() {
      x = 2;
    }
  }

В настройках проверки можно игнорировать присваивание полей. Например, this.x = 2; не будет отмечено, а int y = this.x; будет отмечено.
