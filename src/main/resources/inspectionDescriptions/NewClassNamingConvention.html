Сообщает о классах с именами, которые слишком короткие, слишком длинные или не соответствуют указанному шаблону регулярного выражения.

Пример: если проверка включена для тестов, и указанная минимальная длина имени класса равна 8 (значение по умолчанию), будет выдано предупреждение для следующего тестового класса, так как его имя имеет длину 6, что меньше 8: public class MyTest{}.

Такие классы можно переименовать только с помощью быстрого исправления в редакторе.

Настройка проверки:

Используйте список в разделе Параметры, чтобы указать, какие классы следует проверять. Снимите флажки для классов, которые нужно исключить из проверки.

Для каждого типа класса укажите в соответствующих полях минимальную длину, максимальную длину и регулярное выражение для имени класса. Укажите 0 в полях длины, чтобы пропустить соответствующую проверку.

Регулярные выражения должны быть указаны в стандартном формате java.util.regex.
