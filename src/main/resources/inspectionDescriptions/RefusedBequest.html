Сообщает о методах, которые переопределяются без вызова super метода. Это также известно как отвергнутое наследование. Такие методы могут указывать на нарушение абстракции, что приводит к трудноотслеживаемым ошибкам.

Проверка не сообщает о методах, переопределяющих методы из java.lang.Object (кроме clone()). По соглашению, clone() должен вызывать свой super метод, который вернет объект правильного типа.

Пример 1:


  class A {
    @Override
    public Object clone() {
      // Не вызывает 'super.clone()'
      return new A();
    }
  }

Пример 2:


  interface I {
    default void foo() {}
  }
  class A implements I {
    // Предупреждение о методе выдается, когда отключена
    // опция "Игнорировать 'default' super методы"
    @Override
    public void foo(){}
  }

Настройка проверки:

  • Опция Сообщать только когда super метод аннотирован следующим образом позволяет игнорировать super методы, помеченные аннотациями из предоставленного списка. Вы можете вручную добавлять аннотации в этот список.
  • Опция Игнорировать пустые super методы позволяет игнорировать super методы, которые только выбрасывают исключение или пустые.
  • Опция Игнорировать 'default' super методы позволяет игнорировать default super методы в интерфейсах.
