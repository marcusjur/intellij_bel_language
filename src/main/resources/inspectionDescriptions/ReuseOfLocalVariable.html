Сообщает о локальных переменных, которые "переиспользуются" путем перезаписи их значений, не связанных с их первоначальным назначением.

Такое повторное использование локальных переменных может вызывать путаницу, поскольку предполагаемая семантика локальной переменной может меняться при каждом использовании. Это также может привести к ошибкам, если из-за изменений в коде значение, которое считалось перезаписанным, фактически остается активным. Хорошей практикой является поддержание как можно более короткого жизненного цикла переменных и избегание повторного использования локальных переменных ради краткости.

Пример:


    void x() {
        String s = "one";
        System.out.println("s = " + s);
        s = "two"; //повторное использование локальной переменной 's'
        System.out.println("s = " + s);
    }
