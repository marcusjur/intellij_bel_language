Сообщает о последовательных проверках характеристик java.io.File, таких как:
  • isDirectory()
  • isFile()
  • lastModified()
  • length()
Такие вызовы могут быть заменены пакетным вызовом Files.readAttributes(). Это обычно более производительно, чем несколько отдельных проверок характеристик.

Пример:


  boolean isNewFile(File file, long lastModified) throws IOException {
    return file.isFile() && file.lastModified() > lastModified;
  }

После применения быстрого исправления:


  boolean isNewFile(File file, long lastModified) throws IOException {
    var fileAttributes = Files.readAttributes(file.toPath(), BasicFileAttributes.class);
    return fileAttributes.isRegularFile() && fileAttributes.lastModifiedTime().toMillis() > lastModified;
  }

Если IOException не обрабатывается в текущем контексте, предупреждение не будет показано, но быстрое исправление всё равно доступно.

Обратите внимание, что замена обычно не полностью идентична и должна использоваться с осторожностью. В частности, поведение может отличаться, если файл вообще не существует.

Эта проверка выполняется только если уровень языка проекта или модуля 7 или выше.
