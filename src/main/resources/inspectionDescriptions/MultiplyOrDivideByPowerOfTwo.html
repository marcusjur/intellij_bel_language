Сообщает об умножении целочисленных значений на константы, которые можно представить как степени двойки. Такие выражения можно заменить операциями сдвига вправо или влево для повышения производительности.

Обратите внимание, что эта проверка не актуальна для современных JVM (таких как HotSpot или OpenJ9), поскольку их JIT-компиляторы выполняют эту оптимизацию автоматически. Она может быть полезна только в некоторых встраиваемых системах, где не используется JIT-компиляция.

Пример:


  int y = x * 4;

Рекомендуется использовать быструю замену операции умножения или деления на операцию сдвига:


  int y = x << 2;

Используйте эту опцию, чтобы проверка также сообщала о делении на степени двойки. Обратите внимание, что замена деления на степени двойки операцией сдвига не подходит для отрицательных чисел.
