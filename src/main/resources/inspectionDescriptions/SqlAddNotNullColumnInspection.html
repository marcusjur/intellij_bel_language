Обнаружена попытка создания столбца NOT NULL без значения DEFAULT.

Пример (Microsoft SQL Server):

CREATE TABLE foo (a INT, b  INT)
ALTER TABLE foo ADD c INT NOT NULL;

По умолчанию столбцы содержат значения NULL. В примере мы используем ограничение NOT NULL, чтобы запретить использование NULL в столбце. Если мы запрещаем использование значений NULL, необходимо установить значение DEFAULT, которое SQL будет использовать при создании новых записей. 

ALTER TABLE foo ADD c INT NOT NULL DEFAULT 42;

Вы можете использовать быстрое исправление Добавить значение DEFAULT для быстрого добавления значения по умолчанию.
