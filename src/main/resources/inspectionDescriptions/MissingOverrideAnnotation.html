Сообщает о методах, которые переопределяют методы суперкласса, но не используют аннотацию @java.lang.Override.

Использование аннотации @java.lang.Override улучшает читаемость кода, явно указывая на намерение. Кроме того, компилятор выдаст ошибку, если сигнатура переопределённого метода не соответствует методу суперкласса.

Пример:


    class X {
      public String toString() {
        return "hello world";
      }
    }

После применения быстрого исправления:


    class X {
      @Override
      public String toString() {
        return "hello world";
      }
    }

Настройка проверки:

  • Опция Игнорировать 'equals()', 'hashCode()' и 'toString()' позволяет игнорировать эти методы java.lang.Object: equals(), hashCode() и toString(). Риск того, что эти методы перестанут компилироваться из-за аннотации @Override, относительно низок.
  • Опция Игнорировать методы в анонимных классах позволяет игнорировать методы в анонимных классах.
  • Отключение опции Подсвечивать метод, если не все его переопределения имеют аннотацию '@Override' позволяет выдавать предупреждения только для методов без аннотации @Override, не предупреждая о переопределённых методах в одном или нескольких потомках, где отсутствует аннотация @Override.
