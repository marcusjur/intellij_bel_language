Сообщение о вызове list.remove(index) в цикле с возрастающим счетчиком.

Это подозрительно, так как после удаления список становится короче и следующий элемент пропускается. Простое решение — уменьшить индексную переменную после удаления, но более надежным подходом будет использование итератора или метода removeIf() (Java 8 и выше). Если вы не хотите, чтобы remove() вызывался в цикле несколько раз, можно добавить после него break.

Пример:

  public static void main(String[] args) {
    process(new ArrayList<>(
      Arrays.asList("1", "2", "|", "3", "4")));
  }
  static void process(List<String> list) {
    for (int i = 0; i < list.size(); i++) {
      if (list.get(i).equals("|")) {
        list.remove(i);
        continue;
      }
      System.out.println(list.get(i));
    }
  }

Код выглядит так, будто должен напечатать 1 2 3 4, но на самом деле 3 будет пропущено в выводе.
