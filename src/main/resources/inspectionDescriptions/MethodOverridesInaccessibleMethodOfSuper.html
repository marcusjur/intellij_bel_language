Сообщает о методе, который имеет такую же сигнатуру, как и недоступный метод суперкласса, например private или package-private метод суперкласса из другого пакета.

Такие имена методов могут вводить в заблуждение, поскольку метод в подклассе кажется переопределяющим, но на самом деле скрывает недоступный метод суперкласса. Кроме того, если видимость этого метода в суперклассе изменится в будущем, это может незаметно изменить семантику подкласса или привести к ошибкам компиляции.

Рекомендуется переименовать метод с помощью быстрого исправления.

Пример:


  public class Super {
    private void test() {
    }
  }
  public class Sub extends Super {
    void test() { // Изменение 'Super.test()' на public приведет к ошибке компиляции
                  // Изменение 'Super.test()' на package-private сделает 'Sub.test()' переопределяющим
    }
  }
