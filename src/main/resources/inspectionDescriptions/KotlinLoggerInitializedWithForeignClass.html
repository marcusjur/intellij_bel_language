Сообщает об экземплярах Logger, инициализованных с использованием литерала класса, отличного от класса, в котором находится Logger.

Это может произойти при копировании и вставке кода из другого класса. Это может привести к регистрации событий в неправильной категории и некорректной фильтрации.

С помощью параметров проверки можно указать фабричные классы и методы логгера, которые должна идентифицировать эта проверка.

Пример:


class AnotherService
class MyService {
    private val logger = LoggerFactory.getLogger(AnotherService::class.java)
}

После применения быстрого исправления:


class MyService {
    private val logger = LoggerFactory.getLogger(MyService::class.java)
}
