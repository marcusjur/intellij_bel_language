Сообщается о возможности замены анонимного класса на ссылку на метод.

Обратите внимание, что если анонимный класс преобразуется в несвязанную ссылку на метод, среда выполнения Java может повторно использовать один и тот же объект ссылки на метод при последующих вызовах. С другой стороны, при использовании анонимного класса каждый раз создаётся отдельный объект. Поэтому в редких случаях применение этого быстрого исправления может привести к изменению семантики, например когда экземпляр анонимного класса используется в качестве ключа HashMap.

Пример:


  Runnable r = new Runnable() {
    @Override
    public void run() {
      System.out.println();
    }
  };

После применения быстрого исправления:


  Runnable r = System.out::println;

Используйте опцию Сообщать для интерфейсов без аннотации @FunctionalInterface, чтобы включить эту проверку для интерфейсов, не имеющих аннотации @FunctionalInterface.
