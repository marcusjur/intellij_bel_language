Сообщает о неправильно отформатованных и неверно размещенных сборочных тегах. Утилита go требует, чтобы сборочные теги располагались в определённых местах и следовали специальному синтаксису. Если эти требования не соблюдаются, сборочные теги могут быть проигнорированы или файлы могут быть ошибочно исключены из сборки.

См. Build Constraints на go.dev.

Пример:

package main
// +build ignore
func main() {}

Часть // +build ignore должна находиться перед объявлением пакета. Чтобы исправить эту проблему, вы можете применить быстрое исправление Разместить сборочный тег перед пакетом. После применения быстрого исправления:

// +build ignore
package main
import "fmt"
