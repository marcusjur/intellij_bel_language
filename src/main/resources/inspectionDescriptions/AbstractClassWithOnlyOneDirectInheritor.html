Сообщает об абстрактных (abstract) классах, имеющих только одного прямого наследника. Хотя такие классы могут обеспечивать впечатляющую ясность дизайна в средах с ограниченной памятью или пропускной способностью, они излишне увеличивают общий объем приложения. Рекомендуется рассмотреть возможность объединения абстрактного класса с его наследником.

Эта проверка актуальна для Java ME и других сред с сильно ограниченными ресурсами. Бездумное применение этой проверки может негативно повлиять на ясность кода и дизайн.

Пример:


  abstract class Base {} // будет отмечено в отчёте
  class Inheritor extends Base {}
