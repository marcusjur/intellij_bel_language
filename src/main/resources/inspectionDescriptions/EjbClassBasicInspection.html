Обнаружены следующие нарушения спецификации EJB:
  • Класс не реализует ожидаемый интерфейс (EJBLocalObject, EJBObject и т.д.)
  • В классе не определен публичный конструктор без параметров
  • Использование класса вместо интерфейса или интерфейса вместо класса
  • Метод EJB объявлен как static
  • Stateless Session Bean определяет несколько методов `ejbCreate`
  • Неправильный тип возвращаемого значения в методах Session Bean класса
  • В EJB определен метод `finalize`

Пример:


public class SessionEJB implements java.io.Serializable { // Ошибка: класс должен расширять `SessionBean`
  private SessionEJB() {} // Ошибка: класс должен определять публичный конструктор без параметров
}
