Сообщение о рекурсивных вызовах equals(==).

В Kotlin оператор == сравнивает значения объектов, вызывая метод equals в фоновом режиме. В то время как оператор === сравнивает объекты по ссылке.

Оператор === обычно используется при реализации метода equals. Однако === можно случайно перепутать с ==, что приведет к бесконечной рекурсии.

Пример:


  class X {
      override fun equals(other: Any?): Boolean {
          if (this == other) return true
          return false
      }
  }

После применения быстрого исправления:


  class X {
      override fun equals(other: Any?): Boolean {
          if (this === other) return true
          return false
      }
  }
