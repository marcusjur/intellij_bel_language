Сообщает о вызовах java.sql.Statement.execute() или любых его вариантах, которые используют динамически сконструированные строки в качестве выполняемых запросов.

Сконструированные SQL-запросы являются распространенным источником уязвимостей безопасности. По умолчанию эта проверка игнорирует константы времени компиляции.

Пример:


  ResultSet execute(Statement statement, String name) throws SQLException {
    return statement.executeQuery("select * from " + name); // выдаёт предупреждение
  }

С помощью опций проверки любые поля static final могут рассматриваться как константы. Будьте осторожны, так как при включении этой опции такие строки будут игнорироваться: 


  private static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
