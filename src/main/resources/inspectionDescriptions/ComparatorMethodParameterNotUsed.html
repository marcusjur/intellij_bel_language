Сообщает о проблемах в реализации Comparator.compare() и Comparable.compareTo().

Будут отмечены следующие случаи:

  • Неиспользуемые параметры. Вероятно, это опечатка, сравнение параметра с самим собой или некорректная реализация метода.
  • Метод явно не возвращает 0 для одинаковых элементов. Такой компаратор нарушает контракт и может привести к непредсказуемым результатам при обработке одинаковых элементов. В частности, некоторые операции сортировки могут завершиться с ошибкой.
  • Метод сравнения никогда не возвращает положительное или отрицательное значение. Для соблюдения контракта, если метод возвращает положительное значение, он должен возвращать отрицательное значение при обратном порядке аргументов.
  • Метод сравнения возвращает Integer.MIN_VALUE. Хотя это допускается контрактом, это может привести к ошибкам, так как некоторые вызывающие методы могут некорректно использовать унарный минус для инверсии возвращаемого значения. Отрицательное значение Integer.MIN_VALUE равно Integer.MIN_VALUE.

Пример:


  Comparator<String> lambda =
    (a, b) -> a.length() > b.length()
              ? 0
              : Math.random() > 0.5 ? -1 : 1;
