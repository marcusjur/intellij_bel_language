Сообщает об экземплярах java.util.concurrent.ThreadLocalRandom, которые могут совместно использоваться между потоками.

ThreadLocalRandom не должен использоваться совместно между потоками, так как не является потокобезопасным. Эта проверка может выявлять экземпляры, присвоенные полям, которые используются как аргументы методов, или присвоенные локальным переменным и используемые в анонимных или вложенных классах, поскольку они могут совместно использоваться между потоками.

Использование ThreadLocalRandom обычно должно выглядеть как ThreadLocalRandom.current().nextInt(...) (или nextDouble(...) и т.д.). Когда все использования следуют этой форме, экземпляры ThreadLocalRandom не будут неправильно использоваться несколькими потоками.

Пример:


    class Main {
      void printRandomNumbersAsync() {
        ThreadLocalRandom random = ThreadLocalRandom.current();
        CompletableFuture.supplyAsync(() -> generateNumbers(random))
          .thenAccept(numbers -> System.out.println(Arrays.toString(numbers)));
      }
      private int[] generateNumbers(Random random) {
        return random.ints(1000, 0, 100).toArray();
      }
    }

Используйте эти опции для указания методов, которым могут передаваться экземпляры ThreadLocalRandom в качестве аргументов. Для имен методов можно использовать регулярные выражения.
