Сообщает о не статических внутренних классах, которые реализуют java.io.Serializable, но не определяют поле serialVersionUID.

Без поля serialVersionUID любые изменения в классе приведут к нечитаемости ранее сериализованных версий. Настоятельно рекомендуется, чтобы не статические внутренние классы с Serializable содержали поле serialVersionUID, иначе алгоритм сериализации по умолчанию может привести к несовместимости версий сериализации между компиляторами из-за различий в синтетических методах доступа.

Рекомендуется добавить отсутствующее поле serialVersionUID с помощью быстрого исправления.

Пример:


  class Outer {
      class Inner implements Serializable {}
  }

После применения быстрого исправления:


  class Outer {
      class Inner implements Serializable {
          private static final long serialVersionUID = -7004458730436243902L;
      }
  }

Настройте проверку с помощью следующих параметров:

  • Перечислите классы, наследники которых не должны отмечаться этой проверкой. Это применимо к классам, которые наследуют Serializable от суперкласса, но не используются для сериализации.
  • Следует ли игнорировать анонимные классы с Serializable.
