Сообщает о конструкторах, количество параметров которых превышает указанное максимальное значение. Такие объекты сложно создавать, особенно когда некоторые параметры являются необязательными. Большое количество параметров в конструкторе может указывать на необходимость рефакторинга. Рекомендуется рассмотреть применение паттерна Builder.

Пример:


  public BankAccount(long accountNumber,
                     String owner,
                     double balance,
                     double interestRate) {
    // fields initialization
  }

Настройка проверки:

  • Используйте поле Лимит параметров, чтобы указать максимально допустимое количество параметров в конструкторе.
  • Используйте список Игнорировать конструкторы со следующей видимостью, чтобы указать, какие конструкторы с определённой видимостью следует исключить из проверки.
