Сообщает о методах, которые можно безопасно объявить как static. Объявление методов как static, где это возможно, уменьшает потребление памяти и улучшает качество кода.

Метод может быть объявлен как static, если он:

  • не является synchronized, native или abstract,
  • не обращается к нестатическим методам и полям содержащего класса,
  • не переопределяет и не будет переопределен в подклассах.

Проверка настраивается следующими параметрами:

  • Сообщать только о private и final методах для повышения производительности проверки.
  • Игнорировать пустые методы.
  • Игнорировать методы по умолчанию в интерфейсах при использовании Java 8 или выше.
  • Заменять при быстром исправлении ссылки на экземпляр класса на ссылки на сам класс при вызове метода, объявленного как static, то есть вызов myClass.m() будет заменен на MyClass.m().
