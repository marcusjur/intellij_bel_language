Сообщает об inline-функциях с ненулевым расширяющим получателем, которые не используют тот факт, что расширяющий получатель не может быть null.

До Kotlin 1.2 вызовы inline fun с гибким расширяющим получателем, допускающим null (платформенный тип с неизвестной nullability), не включали проверки на null в байткоде. Начиная с Kotlin 1.2, проверки на null включены в байткод (см. KT-12899).

Следовательно, в версиях до Kotlin 1.2 функции, не использующие тот факт, что расширяющий получатель не может быть null, были потенциально опасны, и рекомендуется сделать такие функции с получателем, допускающим null.

Пример:


  inline fun String.greet() {
      println("Hello, $this!")
  }
  fun main() {
      // `System.getProperty` возвращает непредставимый тип `String!`
      val user = System.getProperty("user.name")
      user.greet()
  }

После применения быстрого исправления:


  inline fun String.greet() {
      println("Hello, $this!")
  }
  fun main() {
      // `System.getProperty` возвращает непредставимый тип `String!`
      val user = System.getProperty("user.name")
      user.greet()
  }

Эта проверка выполняется только если уровень языка Kotlin для проекта или модуля ниже 1.2.
