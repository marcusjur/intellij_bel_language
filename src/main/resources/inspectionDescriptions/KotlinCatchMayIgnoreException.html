Сообщает о пустых блоках catch или блоках, которые могут игнорировать исключения.

Хотя иногда это делается намеренно, пустые блоки catch могут усложнить отладку. Кроме того, игнорирование параметра catch может быть ошибкой.

Эта проверка не выдает предупреждения для параметров catch, названных ignore, ignored или _.

Вы можете использовать быстрое исправление, чтобы изменить имя исключения на _.

Пример:


  try {
    throwingMethod()
  } catch (ex: IOException) {
  }

После применения быстрого исправления:


  try {
    throwingMethod()
  } catch (_: IOException) {
  }
Используйте опцию Не выдавать предупреждения, когда блок 'catch' содержит комментарии, чтобы игнорировать блоки catch, содержащие комментарии.
