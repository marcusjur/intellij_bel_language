Сообщает о случаях, когда можно использовать вызовы Math.max() или Math.min() вместо ручного вычисления минимального или максимального значения двух чисел.

Пример:


  public int min(int a, int b) {
    return b < a ? b : a;
  }

После применения быстрого исправления:


  public int min(int a, int b) {
    return Math.min(a, b);
  }

Используя опцию Отключить для float и double, можно отключить эту проверку для типов double и float. Это полезно, поскольку быстрое исправление может слегка изменить семантику типов float/ double при обработке NaN. Тем не менее, если не учитывать NaN, это исправление в большинстве случаев фактически исправляет трудноуловимые ошибки.
