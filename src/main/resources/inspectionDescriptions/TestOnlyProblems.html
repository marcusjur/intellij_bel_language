Сообщает о методах и классах в производственном коде, использующих аннотации @TestOnly и @VisibleForTesting. Также сообщает о случаях применения @TestOnly и @VisibleForTesting к одному и тому же элементу.

Проблемы не отображаются, если метод или класс вызывается из следующих мест:

  • Код в папке Test Sources
  • Тестовые классы (JUnit/TestNG)
  • Другой метод с аннотацией @TestOnly

Пример (в производственном коде):


  @TestOnly
  fun foo() { ... }
  fun main () {
    foo()
  }
