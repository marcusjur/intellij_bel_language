Сообщение о недопустимом целевом каталоге в командах ADD и COPY.

Согласно спецификации Dockerfile, если указано несколько источников, то целевой путь должен быть каталогом и заканчиваться косой чертой '/'. В противном случае сборка Docker завершится с ошибкой.

Пример:


  # Все следующие команды завершатся ошибкой
  ADD textA.txt textB.txt relativeDir
  ADD ["binaryA.jar", "binary2.jar", "destination"]
  COPY text3.txt text4.txt /absolute/path

После применения быстрого исправления:


  ADD textA.txt textB.txt relativeDir/
  ADD ["binaryA.jar", "binary2.jar", "destination/"]
  COPY text3.txt text4.txt /absolute/path/
