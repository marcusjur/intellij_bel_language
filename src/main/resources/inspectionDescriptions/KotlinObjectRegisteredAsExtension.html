Регистрация Kotlin-объекта в качестве расширения плагина.

Жизненным циклом расширений управляет платформа IntelliJ. Использование Kotlin-объектов для регистрации расширений может привести к созданию ненужных экземпляров расширений и сделать невозможным корректное удаление плагина.

Пример

Регистрация расширения:


<annotator language="myLang" implementationClass="com.example.MyAnnotator"/>

Реализация расширения:


// Неправильно:
object MyAnnotator : Annotator {
  ...
}
// Правильно:
class MyAnnotator : Annotator {
  ...
}
