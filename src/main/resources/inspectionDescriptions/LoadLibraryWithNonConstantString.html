Сообщает о вызовах java.lang.System.loadLibrary(), java.lang.System.load(), java.lang.Runtime.loadLibrary() и java.lang.Runtime.load(), которые используют динамически сконструированные строки в качестве имен библиотек.

Динамически сконструированные имена библиотек часто являются источником уязвимостей безопасности. По умолчанию эта проверка игнорирует константы времени компиляции.

Пример:


  void test(int i) {
    System.loadLibrary("foo" + i);
  }

В настройках проверки можно указать, чтобы любые поля static final рассматривались как константы. Будьте осторожны, так как при включении этой опции такие строки будут игнорироваться: 


  private static final String LIBRARY = getUserInput();
