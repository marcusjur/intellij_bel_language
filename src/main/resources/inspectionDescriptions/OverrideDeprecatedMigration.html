Сообщает об объявлениях с аннотацией @Deprecated, которые вызывают ошибки компиляции начиная с версии 1.9.

Типы мотивации:

  • Необходимость изменения реализации по причинам дизайна/архитектуры
  • Несоответствия в дизайне (различная обработка в разных контекстах)

Дополнительная информация: KT-47902: Не распространять устаревшие методы через переопределение

Быстрое исправление копирует аннотацию @Deprecated из родительского объявления.

Пример:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }
  class Derived : Base() {
      override fun foo() {}
  }

После применения быстрого исправления:


  open class Base {
      @Deprecated("Don't use")
      open fun foo() {}
  }
  class Derived : Base() {
      @Deprecated("Don't use")
      override fun foo() {}
  }

Эта проверка выполняется только если уровень языка Kotlin в проекте или модуле 1.6 или выше.
