Сообщает об операторах и выражениях switch с недостаточным количеством меток case и предлагает переписать их как операторы if и else if.

Пример (минимальное количество веток == 3):


  switch (expression) {
    case "foo" -> foo();
    case "bar" -> bar();
  }

После применения быстрого исправления:


  if ("foo".equals(expression)) {
    foo();
  } else if ("bar".equals(expression)) {
    bar();
  }

Не сообщает об исчерпывающих switch-выражениях (Java 14+) или операторах switch с шаблонами без ветки 'default' (предварительная версия Java 17). Это связано с тем, что при преобразовании switch в if теряется проверка исчерпываемости во время компиляции (что может быть нежелательным).

Настройка проверки:

Используйте поле минимальное количество веток для указания минимального ожидаемого количества меток case.

Используйте опцию не сообщать об операторах switch с шаблонами, чтобы избежать уведомлений об операторах и выражениях switch с ветками шаблонов. Например:


  String result = switch(obj) {
    case String str -> str.trim();
    default -> "none";
  };

Switch следует сохранить даже при наличии только одной ветки с шаблоном, вместо использования оператора instanceof.
