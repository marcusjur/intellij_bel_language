В Java 21 и более ранних версиях сообщается о методах с именем "main", которые не имеют сигнатуры public static void main(String[]). Начиная с предварительной версии Java 21, проверка не подсвечивает методы main с модификаторами package-private, protected или методы экземпляра (также без параметров). Кроме того, сообщается о методах main, находящихся в анонимных или локальных классах. Анонимные и локальные классы не имеют полного квалифицированного имени, поэтому их невозможно запустить.

Такие методы могут вводить в заблуждение, поскольку метод с именем "main" должен быть точкой входа в приложение.

Пример:


  class Main {
    void main(String[] args) {} //предупреждение здесь из-за отсутствия модификаторов "public static"
  }

Быстрое исправление для переименования таких методов доступно только в редакторе.
