Сообщается о ресурсе java.util.concurrent.locks.Lock, который не был получен в блоке try или не был разблокирован в соответствующем блоке finally. Такие ресурсы могут быть случайно утечены, если исключение возникнет до освобождения ресурса.

Пример:


  lock.lock(); // Будет отмечено из-за отсутствия блока 'finally'
  try {
    doSmthWithLock();
  } catch (IOException e) {
    throw new UncheckedIOException(e);
  }
  lock.unlock();
