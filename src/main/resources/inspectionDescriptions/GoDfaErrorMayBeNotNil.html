Сообщает о случаях, когда переменная может иметь значение nil или неожиданное значение из-за того, что связанная ошибка не была проверена на не-nil, как в случае v, err := foo().

Быстрое исправление исключит функцию из области действия этой проверки.

Пример:


func _() {
    file, err := os.Open("file.txt")
    // здесь пропущена проверка ошибки
    name := file.Name()
    print(name, err)
}

В приведенном примере file может быть nil или иметь неожиданное значение, когда err не равно nil.
