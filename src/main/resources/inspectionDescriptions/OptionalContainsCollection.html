Сообщает об использовании типов java.util.Optional или com.google.common.base.Optional с параметрами массива или коллекции.

В таких случаях предпочтительнее использовать пустой массив или коллекцию для обозначения отсутствия результата.

Пример:


  Optional<List<Integer>> foo() {
    return Optional.empty();
  }

Этот код лучше переписать так:


  List<Integer> foo() {
    return List.of();
  }
