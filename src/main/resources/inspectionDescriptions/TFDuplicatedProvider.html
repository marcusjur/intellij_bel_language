Обнаружены повторяющиеся провайдеры.

Пример проблемы:


provider "kubernetes" {
  config_path = "~/.kube/config"
}
provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
}

Один из способов решения этой проблемы - добавить псевдоним и явно указать его при создании ресурсов в нужном провайдере.

Пример исправления:


  provider "kubernetes" {
  config_path = "~/.kube/config"
}
provider "kubernetes" {
  config_path = "~/kube/kubeconfig-0728633f-2c4a-4d2b-9eb7-bc629d5e7f37"
  alias = "cloud"
}
Техническая поддержка: Плагин Terraform и HCL
