Сообщает о классах, которые переопределяют equals(), но не переопределяют hashCode() (или наоборот). Также сообщает об объявлениях объектов, которые переопределяют equals() или hashCode().

Это может привести к некорректному поведению при добавлении класса в Collection

Пример:


  class C1 {
      override fun equals(other: Any?) = true
  }
  class C2 {
      override fun hashCode() = 0
  }
  object O1 {
      override fun equals(other: Any?) = true
  }
  object O2 {
      override fun hashCode() = 0
  }

Быстрое исправление добавит переопределение метода equals() или hashCode() для классов и удалит эти методы для объектов:


   class C1 {
       override fun equals(other: Any?) = true
       override fun hashCode(): Int {
           return javaClass.hashCode()
       }
   }
   class C2 {
       override fun hashCode() = 0
       override fun equals(other: Any?): Boolean {
           if (this === other) return true
           if (javaClass != other?.javaClass) return false
           return true
       }
   }
   object O1 {
   }
   object O2 {
   }
