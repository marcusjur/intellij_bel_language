Защитный монитор представляет собой аннотацию @GuardedBy на методе или поле экземпляра, которая ссылается на static поле. Использование static для защиты нестатических элементов может привести к чрезмерной конкуренции за блокировку, поскольку доступ к любому заблокированному полю в любом экземпляре объекта будет препятствовать одновременному доступу к полям во всех экземплярах объекта.

Пример:


    private static ReadWriteLock lock = new ReentrantReadWriteLock(); //статическая защита поля
    private Object state;
    @GuardedBy("lock")
    public void bar() {
        state = new Object();
    }

Поддерживаемые аннотации @GuardedBy:

  • net.jcip.annotations.GuardedBy
  • javax.annotation.concurrent.GuardedBy
  • org.apache.http.annotation.GuardedBy
  • com.android.annotations.concurrency.GuardedBy
  • androidx.annotation.GuardedBy
  • com.google.errorprone.annotations.concurrent.GuardedBy
