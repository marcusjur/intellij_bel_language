Сообщает о переопределении метода Object.finalize().

Согласно документации Object.finalize():

Механизм финализации является проблематичным сам по себе. Финализация может вызвать проблемы с производительностью, взаимоблокировки и зависания. Ошибки в финализаторах могут привести к утечке ресурсов; финализацию невозможно отменить, если она больше не нужна; и порядок вызова метода finalize для разных объектов не определён. Более того, нет никаких гарантий относительно времени финализации. Метод finalize вызывается для финализуемых объектов (если таковые имеются) только после неопределённой задержки.

Настройка проверки:

  • Опция игнорировать обычные реализации 'finalize()' позволяет игнорировать реализации finalize() с пустым телом метода или телом, содержащим только условный оператор if с условием, вычисляемым как false во время компиляции. По соображениям производительности может быть полезно переопределить нетривиальный метод finalize() пустой реализацией в подклассе. Пустая финальная реализация finalize() также может использоваться для предотвращения переопределения в подклассах.
