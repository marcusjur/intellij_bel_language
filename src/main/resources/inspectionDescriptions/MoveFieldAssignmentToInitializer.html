Предлагается заменить инициализацию полей с помощью присваивания на инициализацию при объявлении полей.

Проблема отмечается только когда присваивание находится в инициализаторе экземпляра или статическом инициализаторе, и когда объединение с объявлением поля считается безопасным. В других случаях (например, при присваивании в конструкторе) быстрое исправление не подсвечивается, так как изменение может повлиять на семантику кода.

Пример:


  class MyClass {
    static final int intConstant;
    static {
      intConstant = 10;
    }
  }
Быстрое исправление переместит присвоенное значение в инициализатор поля и удалит инициализатор класса, если это возможно: 

  class MyClass {
    static final int intConstant = 10;
  }
Начиная с версии 2017.2
