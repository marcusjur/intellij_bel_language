Сообщает о вызовах Collection<T>.count().

Этот вызов функции может быть заменен на .size.

Форма .size гарантирует, что операция выполняется за O(1) и не выделяет дополнительных объектов, тогда как count() можно спутать с Iterable<T>.count(), который выполняется за O(n) и производит выделение памяти.

Пример:

  fun foo() {
      var list = listOf(1,2,3)
      list.count() // Заменяемый вызов 'count()'
  }

После применения быстрого исправления:

  fun foo() {
      var list = listOf(1,2,3)
      list.size
  }
