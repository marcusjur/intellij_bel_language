Сообщать о нефильтрованных циклах for-in.

Использование этой конструкции обрабатывает не только собственные свойства объекта, но и свойства его прототипа. В определённых случаях это может привести к неожиданным результатам, например в служебных методах для копирования или изменения всех свойств, или когда прототип Object может быть ошибочно модифицирован. Например, следующий код выведет 42 и myMethod: 


Object.prototype.myMethod = function myMethod() {};
let a = { foo: 42 };
for (let i in a) {
  console.log(a[i]);
}

Рекомендуется заменить весь цикл на метод Object.keys() или добавить проверку hasOwnProperty(). После применения быстрого исправления код будет выглядеть так:


for (let i in a) {
  if (a.hasOwnProperty(i)) {
    console.log(a[i]);
  }
}
