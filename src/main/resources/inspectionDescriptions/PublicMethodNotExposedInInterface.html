Сообщает о методах public, которые не объявлены в интерфейсах.

Объявление всех public методов через интерфейсы важно для поддержания слабой связанности, что может быть необходимо для некоторых компонентно-ориентированных стилей программования.

Пример:


interface Person {
  String getName();
}
class PersonImpl implements Person {
  private String name;
  // Правильно: метод объявлен в интерфейсе
  @Override
  public String getName() {
    return name;
  }
  // Предупреждение: метод public,
  // но не объявлен в интерфейсе
  public void setName() {
    this.name = name;
  }
}

Используйте список Игнорировать при наличии аннотаций для указания специальных аннотаций. Проверка будет игнорировать методы, помеченные одной из этих аннотаций.

Используйте опцию Игнорировать, если класс не реализует нестандартные интерфейсы, чтобы пропускать методы в классах, которые не реализуют никаких интерфейсов проекта.
