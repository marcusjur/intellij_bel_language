Сообщает об исключениях, которые выбрасываются внутри блока catch без "обёртывания" пойманного исключения.

Когда исключение выбрасывается в ответ на другое исключение, обёртывание исходного исключения предотвращает потерю важной контекстной информации, такой как кадры стека и номера строк.

Пример:


  ...
  catch (IOException e) {
    closeAllConnections();
    throw new ConnectException("Connection problem."); // Предупреждение: 'throw' в блоке 'catch' игнорирует пойманное исключение 'e'
  }

Настройка проверки:

  • Используйте опцию Игнорировать, если используется результат метода исключения, чтобы указать, должна ли проверка игнорировать исключения, аргументом которых является результат вызова метода исходного исключения, например getMessage().
  • Используйте опцию Игнорировать, если выброшенное исключение не может обернуть исключение, чтобы игнорировать операторы throw, выбрасывающие исключения без конструктора, принимающего параметр типа Throwable.
