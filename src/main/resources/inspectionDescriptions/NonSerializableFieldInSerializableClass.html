Сообщает о несериализуемых полях в классах, реализующих java.io.Serializable. Такие поля приведут к исключению времени выполнения при сериализации объекта.

Не сообщается о полях, объявленных как transient или static, а также о полях классов, определяющих метод writeObject.

Эта проверка предполагает, что поля типов java.util.Collection и java.util.Map являются Serializable, если только их объявленный тип не является несериализуемым.

Пример:


  class NonSerializableClass {}
  public class SerializableClass implements Serializable {
      NonSerializableClass clazz; // Предупреждение: несериализуемое поле 'clazz' в сериализуемом классе
      static NonSerializableClass staticClazz; // Нет предупреждения
  }

Настройте проверку с помощью следующих параметров:

  • Укажите классы, наследники которых не должны проверяться. Это применимо к классам, которые наследуют Serializable от суперкласса, но не используются для сериализации.
  • Укажите аннотации, при наличии которых поля будут игнорироваться при проверке.
  • Следует ли игнорировать поля, инициализованные анонимными классами.
