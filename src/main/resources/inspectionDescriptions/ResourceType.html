Эта проверка анализует вызовы Android API с различными поддерживаемыми аннотациями (такими как RequiresPermission или UiThread) и отмечает любые вызовы API, которые не используются правильно согласно указанным аннотациям.

Примеры ошибок, отмечаемых этой проверкой:
  • Передача целочисленного ресурса неверного типа (например, R.string) в API, требующий другой тип (например, R.dimen).
  • Отсутствие вызова переопределённого метода (через super) в методах, где это необходимо
  • Вызов методов, требующих разрешения, которые не объявлены в манифесте
  • Передача ссылки на цветовой ресурс методу, который ожидает целочисленное RGB-значение.
...и многое другое.

Для получения дополнительной информации см. документацию по адресу http://developer.android.com/tools/debugging/annotations.html
