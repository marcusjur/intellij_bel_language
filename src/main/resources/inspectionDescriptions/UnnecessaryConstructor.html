Обнаружение ненужных конструкторов.

Конструктор считается ненужным, если он является единственным конструктором класса, не имеет параметров, имеет тот же модификатор доступа, что и его класс, и не выполняет никакой инициализации, кроме явного или неявного вызова конструктора суперкласса без аргументов. Такой конструктор можно безопасно удалить, поскольку он будет автоматически сгенерирован компилятором, даже если не указан явно.

Пример:


  public class Foo {
    public Foo() {}
  }

После применения быстрого исправления:


  public class Foo {}

В настройках проверки можно настроить игнорирование ненужных конструкторов с аннотациями.
