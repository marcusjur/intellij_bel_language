Сообщать об использовании автоматических модулей в директиве requires.

Автоматические модули ненадежны, так как они могут зависеть от типов в пути к классам, а их имена и экспортуемые пакеты могут измениться при преобразовании в явные модули.

Соответствует опциям Javac -Xlint:requires-automatic и -Xlint:requires-transitive-automatic. Первая опция улучшает понимание того, когда используются автоматические модули. Вторая опция предупреждает авторов модулей о том, что установка неявной читаемости для автоматического модуля подвергает пользователей модуля риску.

Пример:


  //module-info.java
  module org.printer {
    requires transitive drivers.corp.org; // Сообщение при автоматическом модуле 'drivers.corp.org'
  }

Используйте опцию Подсвечивать только транзитивные зависимости, чтобы получать предупреждения только о транзитивных зависимостях.
