Сообщение о попытке присвоить значение сгенерированному столбцу в операторе INSERT. Сгенерированные столбцы можно читать, но нельзя напрямую записывать в них значения.

Пример (PostgreSQL):

CREATE TABLE foo
(
   col1 INT,
   col2 INT GENERATED ALWAYS AS (col1 + 1) STORED
);
INSERT INTO foo(col1, col2) VALUES (1, 2);

Вы не можете вставить значение 2 в столбец col2, так как этот столбец является сгенерированным. Чтобы этот скрипт работал корректно, вы можете заменить 2 на DEFAULT.

INSERT INTO foo(col1, col2) VALUES (1, DEFAULT);
