Сообщает о вызовах java.sql.Connection.prepareStatement(), java.sql.Connection.prepareCall() или любых их вариантах, которые используют динамически сконструированные строки в качестве подготавливаемых выражений.

Сконструированные SQL-запросы являются распространенным источником уязвимостей безопасности. По умолчанию эта проверка игнорирует константы времени компиляции.

Пример:


  String bar() { return "bar"; }
  Connection connection = DriverManager.getConnection("", "", "");
  connection.("SELECT * FROM user WHERE name='" + bar() + "'");
Используя настройки проверки, можно рассматривать любые поля static final как константы. Будьте осторожны, так как при включении этой опции такие строки будут игнорироваться: 

  static final String SQL = "SELECT * FROM user WHERE name='" + getUserInput() + "'";
