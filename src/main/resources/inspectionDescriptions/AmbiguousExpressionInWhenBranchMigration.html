Сообщает о неоднозначных логических выражениях в ветвях оператора when, которые будут вызывать ошибки компиляции в Kotlin 1.8 и выше.


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      this in (4..7) -> true // неоднозначно
      else -> false
  }

После применения быстрого исправления:


  fun Int.matches(strict: Boolean): Boolean = when (strict) {
      true -> this == 6
      (this in (4..7)) -> true // обернуто в круглые скобки
      else -> false
  }

Проверка доступна для уровня языка Kotlin начиная с версии 1.7.
