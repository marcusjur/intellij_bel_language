Обнаруживает не-abstract тестовые классы, не содержащие тестовых методов. Такие тестовые классы обычно указывают на незавершенный код или могут быть остатками после рефакторинга, которые следует удалить.

Пример:


  public class CrucialTest {
    @Before
    public void setUp() {
      System.out.println("setting up");
    }
  }

Используйте опцию Игнорировать тестовые классы с тестовыми методами в суперклассах, чтобы исключить из проверки тестовые классы, у которых есть тестовые методы в суперклассах.
