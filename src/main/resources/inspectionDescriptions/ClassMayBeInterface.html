Сообщает об abstract классах, которые могут быть преобразованы в интерфейсы.

Предпочтительнее использовать интерфейсы вместо классов, поскольку Java не поддерживает множественное наследование классов, но класс может реализовывать несколько интерфейсов.

Класс может быть преобразован в интерфейс, если у него нет суперкласса (кроме Object), есть только поля public static final, методы public abstract и внутренние классы public.

Пример:


abstract class Example {
    public static final int MY_CONST = 42;
    public abstract void foo();
}
class Inheritor extends Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

После применения быстрого исправления:


interface Example {
    int MY_CONST = 42;
    void foo();
}
class Inheritor implements Example {
    @Override
    public void foo() {
        System.out.println(MY_CONST);
    }
}

Настройка проверки:

Опция Сообщать о классах с неабстрактными методами при использовании Java 8 позволяет обнаруживать только те классы, чьи static и неабстрактные методы могут быть преобразованы в default методы (применимо только для уровня языка 8 и выше).
