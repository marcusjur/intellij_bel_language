Сообщает о полях с именами, которые слишком короткие, слишком длинные или не соответствуют указанному шаблону регулярного выражения.

Пример: если проверка включена для констант, и минимальная указанная длина имени поля равна 5 (значение по умолчанию), будет выдано предупреждение для следующей константы, так как длина её имени равна 3, что меньше 5: public static final int MAX = 42;.

Быстрое исправление для переименования таких полей доступно только в редакторе.

Настройка проверки:

Используйте список в разделе Параметры, чтобы указать, какие поля следует проверять. Снимите флажки для тех полей, которые нужно исключить из проверки.

Для каждого типа поля укажите минимальную длину, максимальную длину и регулярное выражение в соответствующих полях ввода. Укажите 0 в полях длины, чтобы пропустить соответствующую проверку.

Регулярные выражения должны быть указаны в стандартном формате java.util.regex.
