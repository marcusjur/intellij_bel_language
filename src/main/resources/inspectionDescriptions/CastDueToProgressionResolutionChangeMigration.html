Аргументы требуют явного преобразования для правильного разрешения перегруженного вызова функции. Текущее предупреждение компилятора (доступное с Kotlin 1.6.20) станет ошибкой в Kotlin 1.8.

Прогрессии и типы диапазонов (kotlin.ranges) будут реализовывать интерфейс Collection начиная с версии 1.9 и выше. Это изменение повлияет на разрешение перегруженных функций. Например, в приведенном ниже примере вызов test(1..5) будет разрешен как test(t: Any) в Kotlin 1.8 и более ранних версиях, но как test(t: Collection<*>) в Kotlin 1.9 и более поздних версиях. 


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }
  fun invoke() {
    test(1..5) // IntRange станет Collection в версии 1.9
  }

Предлагаемое быстрое исправление фиксирует поведение, специфичное для компилятора версии 1.8 и ранее:


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }
  fun invoke() {
    test(1..5) // Разрешается как 'test(t: T)' до Kotlin 1.9
  }

После применения быстрого исправления:


  fun test(t: Any) { }
  fun test(t: Collection<*>) { }
  fun invoke() {
    test((1..5) as Iterable<Int>) // Разрешается как 'test(t: T)' в Kotlin 1.9
  }

Проверка уровня языка Kotlin доступна начиная с версии 1.6.
