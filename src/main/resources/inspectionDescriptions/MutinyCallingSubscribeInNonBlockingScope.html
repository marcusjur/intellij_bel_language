Сообщать о вызовах subscribe() в "reactive" методах.

Методы, возвращающие Publisher (включая Uni и Multi), не должны напрямую вызывать метод subscribe(), так как это может нарушить цепочку реактивных вызовов.

Вместо использования subscribe() рекомендуется использовать операторы композиции, такие как flatMap(), merge() и другие.

Пример:


  Multi<String> stringMulti(){
    Multi<String> multi = Multi.createFrom().item("abc");
    multi.subscribe(); // <- Блокирующий вызов 'subscribe' в неблокирующем контексте
    return multi;
  }
