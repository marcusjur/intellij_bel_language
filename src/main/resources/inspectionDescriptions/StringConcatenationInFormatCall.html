Обнаружена конкатенация нестатических строк, используемых в качестве аргументов форматования.

Хотя иногда это делается намеренно, обычно это является неправильным использованием методов форматования и может привести к проблемам безопасности, когда переменные в конкатенированных строках содержат специальные символы, такие как %.

Кроме того, иногда это может быть результатом ошибочного использования + вместо , при передаче аргументов форматования строки.

Пример:


  static String formatGreeting(String userName) {
    return String.format("Hello, " + userName);
  }

В данном случае userName будет интерпретироваться как часть строки форматования, что может привести к IllegalFormatException (например, если userName равен "%") или к чрезмерному использованию памяти (например, если userName равен "%2000000000%"). Этот вызов следует заменить на String.format("Hello, %s", userName);.

Эта проверка анализует вызовы методов форматования для классов java.util.Formatter, java.lang.String, java.io.PrintWriter или java.io.PrintStream.
