Сообщает о избыточных вызовах asSequence(), которые никогда не приводят к положительному влиянию на производительность.

asSequence() ускоряет обработку коллекций с несколькими операциями, так как выполняет операции отложенно и не создает промежуточные коллекции.

Однако, если сразу после asSequence() следует терминальная операция (например, toList()), это не даст никакого положительного эффекта для производительности.

Пример:


  fun test(list: List<String>) {
      list.asSequence().last()
  }

После применения быстрого исправления:


  fun test(list: List<String>) {
      list.last()
  }
