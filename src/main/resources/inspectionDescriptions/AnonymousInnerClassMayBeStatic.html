Сообщается, что анонимный класс может быть безопасно заменен на внутренний класс с модификатором static. Анонимный класс может быть преобразован во внутренний класс с модификатором static, если он явно не ссылается на внешний экземпляр или локальный класс из окружающего метода.

Внутренние классы с модификатором static не сохраняют неявную ссылку на внешний экземпляр. Это предотвращает распространенные утечки памяти и уменьшает объем памяти, используемой каждым экземпляром класса.

Начиная с Java 18, только сериализуемые анонимные классы сохраняют неявную ссылку на внешний экземпляр, даже если эта ссылка не используется. Поэтому, если уровень языка модуля установлен на Java 18 или выше, эта проверка будет выполняться только для сериализуемых классов.

Быстрое исправление извлечет анонимный класс в именованный внутренний класс с модификатором static.

Пример:


  void sample(){
    Thread thread = new Thread(new Runnable() {
      @Override
      public void run() {
      }
    });
  }

После применения быстрого исправления:


  void sample() {
    Thread thread = new Thread(new Task());
  }
  private static class Task implements Runnable {
    @Override
    public void run() {
    }
  }
