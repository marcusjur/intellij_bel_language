Сообщает о вызовах методов в условиях циклических конструкций. В средах с сильно ограниченными ресурсами такие вызовы могут негативно влиять на производительность.

Бездумное применение этой проверки может отрицательно сказаться на читаемости кода и его архитектуре. Эта проверка предназначена для Java ME и других сред с сильно ограниченными ресурсами.

Пример:


  String s = "example";
  for (int i = 0; i < s.length(); i++) {
    System.out.println(s.charAt(i));
  }

После применения быстрого исправления:


  String s = "example";
  int length = s.length();
  for (int i = 0; i < length; i++) {
    System.out.println(s.charAt(i));
  }

Используйте эту опцию, чтобы игнорировать вызовы стандартных методов итерации Java (например, Iterator.hasNext()) и известных методов с побочными эффектами (например, Atomic*.compareAndSet).
