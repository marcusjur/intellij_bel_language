Сообщает о доступе к нефинальным полям как в synchronized, так и в не-synchronized контекстах. Проверка игнорирует доступ к полям volatile, а также доступ в конструкторах и инициализаторах.

Такой "частично синхронизованный" доступ обычно является результатом ошибки программования и может привести к непредвиденным несоответствиям в структурах данных.

Пример:


  public class Program {
    Console console; // Предупреждение: поле 'console' доступно как в синхронизованном, так и в несинхронизованном контексте
    public synchronized void execute() {
      console.print("running");
    }
    public void check() {
      console.check();
    }
  }

Используйте эту опцию, чтобы указать, следует ли также учитывать простые геттеры и сеттеры как точки доступа.
