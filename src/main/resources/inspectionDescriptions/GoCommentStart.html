Сообщать о комментариях, которые не начинаются с имени экспортуемого элемента.

Согласно Comment Sentences на github.com/golang, принято начинать комментарий с имени экспортуемого элемента.

Пример:

// представляет запрос на выполнение команды.
type Request struct {}

Комментарий начинается с описания структуры, а не с её имени. Чтобы соблюсти правило, вы можете применить быстрое исправление добавить префикс в комментарий. После применения быстрого исправления комментарий будет выглядеть так:

// Request представляет запрос на выполнение команды.
type Request struct {} // лучше
