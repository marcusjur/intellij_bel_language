Сообщает об экземплярах AutoCloseable, которые не используются в операторе try-with-resources, также известном как автоматическое управление ресурсами.

Также сообщает о стиле "открытие ресурса перед/внутри try и закрытие в finally", который использовался до появления try-with-resources. При разработке на Java 7 и выше эта проверка предназначена для замены всех проверок на открытые, но небезопасно закрытые ресурсы.

Пример:


  private static void foo() throws IOException {
    InputStream profile = Thread.currentThread().getContextClassLoader().getResourceAsStream("/someFile");
    System.out.println(profile.read());
  }

Проверка настраивается следующими параметрами:

  • Список подклассов AutoCloseable, которые не требуют закрытия и должны игнорироваться этой проверкой.
    Примечание: Даже если указан java.util.stream.Stream для игнорирования, проверка всё равно будет сообщать о потоках, возвращаемых методами lines(), walk(), list() и find() класса java.nio.file.Files. Эти потоки содержат связанные ресурсы ввода-вывода, которые требуют закрытия.
  • Список методов, возвращающих AutoCloseable, которые следует игнорировать при вызове.
  • Следует ли игнорировать AutoCloseable для результатов вызовов методов. Когда эта опция включена, результаты фабричных методов также будут игнорироваться.
  • Должна ли проверка сообщать о случаях, когда экземпляр AutoCloseable передается как аргумент вызова метода. Если эта опция включена, проверка будет предполагать, что ресурс закрывается в вызываемом методе. Блоки finally и вызовы методов в аргументах AutoCloseable, содержащие 'close' в имени, не игнорируются.
  • Следует ли игнорировать ссылки на методы конструкторов классов ресурсов.
  • Следует ли игнорировать методы, возвращающие ресурсы и начинающиеся с 'get'. Это может уменьшить количество ложных срабатываний, так как большинство геттеров не передают владение ресурсами, и точки их вызова не отвечают за закрытие ресурсов.
