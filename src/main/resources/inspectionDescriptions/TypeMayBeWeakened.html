Инспекция предлагает изменить типы переменных и возвращаемые типы методов на более абстрактные (более слабые). Это делает код более абстрактным и, следовательно, более пригодным для повторного использования.

Пример:


  // Тип параметра может быть ослаблен до java.util.List
  void processList(ArrayList<String> list) {
    if (list.isEmpty()) return;
    System.out.println("Processing");
    for (String s : list) {
      System.out.println("String: " + s);
    }
  }

Включите флажок Ослаблять только до интерфейсов, чтобы сообщать о проблемах только в тех случаях, когда тип может быть ослаблен до интерфейса.

Включите флажок Не предлагать ослабление для переменных, объявленных как 'var', чтобы предотвратить отображение предложений для локальных переменных, объявленных с помощью ключевого слова 'var' (Java 10+)

Стоп-классы предназначены для предотвращения ослабления типов ниже определённого уровня, даже если это технически возможно. В некоторых случаях это может улучшить читаемость кода.
