Исключения, выброшенные с помощью оператора throw, всегда могут быть перехвачены включающим их оператором try.

Использование оператора throw в качестве "goto" для изменения локального потока управления может вызвать путаницу и привести к снижению производительности.

Пример:


  try {
    if (!Files.isDirectory(PROJECTS)) {
        throw new IllegalStateException("Directory not found."); // Предупреждение: 'throw' перехватывается включающим оператором 'try'
    }
    ...
  } catch (Exception e) {
      LOG.error("run failed");
  }

Используйте опцию игнорировать повторно выброшенные исключения, чтобы пропускать повторно выброшенные исключения.
