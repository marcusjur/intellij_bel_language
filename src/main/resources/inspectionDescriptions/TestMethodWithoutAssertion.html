Обнаружен тестовый метод, не содержащий утверждений (assertions). Такие методы могут указывать на неполноту или слабость тестового случая.

Пример:


  public class ExtensiveTest {
    @Test
    public void testAlive() {
      System.out.println("nothing");
    }
  }

Настройка проверки:

  • Используйте таблицу для указания комбинаций полных имен классов и регулярных выражений для методов, которые должны рассматриваться как утверждения. Имена классов также соответствуют подклассам.
  • Используйте опцию считать ключевое слово 'assert' утверждением, чтобы указать, следует ли рассматривать Java-утверждения с ключевым словом assert как проверочные утверждения.
  • Используйте опцию игнорировать тестовые методы с объявленными исключениями, чтобы пропускать тестовые методы, объявляющие исключения. Это полезно, когда ваши тесты выбрасывают исключения при неудаче и поэтому не требуют дополнительных утверждений.
