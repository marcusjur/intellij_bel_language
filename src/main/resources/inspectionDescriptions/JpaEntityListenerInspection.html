Обнаружены следующие ошибки спецификации JPA:
  • Класс слушателя сущности без публичного конструктора без параметров
  • Методы слушателя сущности с некорректной сигнатурой

Пример:


  @Entity
  @EntityListeners(value = JavaEntityListener.class)
  class JavaEntity { ... }
  class JavaEntityListener {
    @PostLoad
    void postLoad(String parameter) { ... } // Ошибка: метод 'postLoad' должен принимать параметр типа 'Object'
  }
