Сообщает об использовании класса, когда его видимость ниже, чем видимость использующего его члена в сигнатуре поля или метода. Хотя такие члены являются допустимыми в Java, их нельзя использовать за пределами области видимости типа класса, на который они ссылаются.

Пример:


  public class Parent {
    public Child getChild() {
      return new Child();
    }
    private class Child {}
  }

Кроме того, в Java 9 и более поздних версиях модули могут скрывать некоторые свои классы от других модулей, не экспортуя их пакеты. Однако если члены, являющиеся частью экспортуемого API, ссылаются на неэкспортуемые классы в своей сигнатуре, такие члены нельзя использовать за пределами модуля.

Настройка проверки:

  • Используйте опцию Сообщать о неэкспортуемых классах, раскрытых в API модуля (Java 9+), чтобы получать уведомления о членах API модуля, которые раскрывают неэкспортуемые классы.
    Обратите внимание, что для этой опции уровень языка проекта или модуля должен быть 9 или выше.
  • Используйте опцию Сообщать о недоступных классах, раскрытых в публичном API, чтобы получать уведомления о публичных членах, которые раскрывают классы с более низкой видимостью.
  • Используйте опцию Сообщать о private классах, раскрытых в package-local API, чтобы получать уведомления о package-local членах, которые раскрывают private классы.
