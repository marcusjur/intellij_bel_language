Сообщает об отсутствующих или избыточных подсказках при создании UastHintedVisitorAdapter.

UastHintedVisitorAdapter может принимать подсказки, которые оптимизуют выполнение UAST-визитора, посещая только элементы классов, указанных в списке подсказок.

Например, если визитор реализует метод visitCallExpression, но не включает UCallExpression в подсказках, этот метод не будет выполнен.

И наоборот, если визитор не реализует метод visitCallExpression, но включает UCallExpression в подсказках, унаследованный метод будет выполняться без необходимости.

Примеры


return UastHintedVisitorAdapter.create(
  language,
  object : AbstractUastNonRecursiveVisitor() {
    override fun visitForExpression(node: UForExpression): Boolean {
      inspectLoopExpression()
      return true
    }
    override fun visitDoWhileExpression(node: UDoWhileExpression): Boolean { // не используется во время выполнения
      inspectLoopExpression()
      return true
    }
    //...
  },
  arrayOf(UForExpression::class.java) // Ошибка: отсутствует UDoWhileExpression::class.java
)


UastHintedVisitorAdapter.create(
  language,
  object : AbstractUastNonRecursiveVisitor() {
    override fun visitForExpression(node: UForExpression): Boolean {
      inspectLoopExpression()
      return true
    }
    //...
  },
  arrayOf(
    UForExpression::class.java,
    UDoWhileExpression::class.java // Ошибка: посещение UDoWhileExpression не реализовано
  )
)
