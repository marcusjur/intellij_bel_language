Отчёт в основном используется для локальных переменных классов-оберток примитивных типов.

В некоторых случаях упаковка может привести к серьезному снижению производительности, особенно в циклах.

Используется эвристическая оценка количества операций упаковки. Например, преобразования внутри цикла учитываются с большим весом.

Пример:

  public void example() {
    Integer value = 12;
    needBox(value);
    for (int i = 0; i < 10; i++) {
      // Использование в цикле становится все более частым
      needPrimitive(value);
    }
  }
  void needPrimitive(int value) {}
  void needBox(Integer value) {}

После применения быстрого исправления:

  public void example() {
    int value = 12;
    needBox(value);
    for (int i = 0; i < 10; i++) {
      // Использование в цикле становится все более частым
      needPrimitive(value);
    }
  }
  void needPrimitive(int value) {}
  void needBox(Integer value) {}
