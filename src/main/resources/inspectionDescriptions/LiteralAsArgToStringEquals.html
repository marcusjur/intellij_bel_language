Обнаружен вызов String.equals() или String.equalsIgnoreCase() со строковым литералом в качестве аргумента.

Некоторые стандарты кодования требуют, чтобы строковый литерал был квалификатором метода equals(), а не аргументом, чтобы минимизовать риск возникновения NullPointerException.

Литерал и выражение можно поменять местами с помощью быстрого исправления.

Пример:


  boolean isFoo(String value) {
    return value.equals("foo");
  }

После применения быстрого исправления:


  boolean isFoo(String value) {
    return "foo".equals(value);
  }
