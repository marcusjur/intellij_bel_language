Сообщать о локальных переменных, которые имеют то же имя, что и поля окружающего класса. Такое именование может привести к случайному использованию переменной вместо поля, даже если совпадение имен было намеренным.

Рекомендуется переименовать переменную с помощью быстрого исправления.

Пример:


  public class Foo {
    public Object foo;
    void bar() {
      Object o = new Object() {
        void baz() {
          Object foo; // Локальная переменная 'foo' скрывает поле класса 'Foo'
        }
      };
    }
  }

Для этой проверки можно настроить следующие параметры:

  1. Игнорировать недоступные поля - игнорировать локальные переменные с именами, совпадающими с невидимыми полями суперкласса (например, если эти поля являются private).
  2. Игнорировать локальные переменные в статическом контексте, скрывающие нестатические поля, - например, когда локальная переменная находится внутри статического метода или внутри метода статического внутреннего класса.
