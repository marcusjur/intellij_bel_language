Сообщать о структурах, инициализованных без указания имен полей. По умолчанию эта проверка активна только при использовании типов, определённых в других пакетах.

При инициализации структур рекомендуется явно указывать имена полей, чтобы обеспечить их корректную адресацию даже при изменении порядка или названий этих полей.

Пример:

_ = io.LimitedReader{nil, 10}

Инициализация LimitedReader будет подсвечена, так как отсутствуют явные имена полей структуры. Вы можете применить быстрое исправление Добавить ключи и удалить нулевые значения. После применения быстрого исправления код будет выглядеть так:

_ = io.LimitedReader{N: 10}

Проверка имеет следующие параметры:

  • Сообщать о типах, определённых в текущем пакете: сообщать о случаях, когда структуры объявляются и инициализуются в одном и том же пакете. Рассмотрим следующий пример.

    d := struct {
  c string
 }{ "dsd "}

  • Сообщать о нетипизованных литералах структур: сообщать о полях без имен, когда тип (структура или интерфейс) опущен. Следующий пример показывает случай с опущенным типом.

    _ = []io.LimitedReader{
 {nil, 10},
 {nil, 20},
 }

    В следующем примере тип присутствует, но избыточен.

    _ = []io.LimitedReader{
 io.LimitedReader{nil, 10},
 io.LimitedReader{nil, 20},
}
