Сообщать о параметрах метода, которые совпадают с именами полей окружающего класса. Такое именование может привести к случайному использованию параметра вместо поля, даже когда предполагалось использование одноименного поля.

Рекомендуется переименовать параметр с помощью быстрого исправления.

Пример:


  class Main {
    private String value;
    public Main(String value) {
      value = value.toUpperCase();
    }
  }

Для этой проверки можно настроить следующие параметры:

  1. Игнорировать для сеттеров свойств - игнорировать параметры в простых сеттерах.
  2. Игнорировать поля суперкласса, невидимые из подклассов - игнорировать private поля в суперклассе, которые не видны для данного метода.
  3. Игнорировать для конструкторов - игнорировать параметры конструкторов.
  4. Игнорировать для абстрактных методов - игнорировать параметры абстрактных методов.
  5. Игнорировать параметры статических методов, скрывающие поля экземпляра - игнорировать параметры static методов, которые скрывают поля экземпляра, а также параметры методов экземпляра в статических внутренних классах, которые скрывают поля экземпляра внешнего класса. Хотя это не является строгим сокрытием, такие параметры все равно могут вызывать путаницу.
