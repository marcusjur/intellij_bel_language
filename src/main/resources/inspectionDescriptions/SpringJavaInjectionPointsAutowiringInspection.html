Сообщает о проблемах автоматического связывания (autowiring) в точках внедрения Spring Bean с аннотациями @Component, @Service и другими.

  • Более одного конкретного ('concrete') типа Bean
  • Отсутствие конкретного ('concrete') типа Bean
  • Отсутствие Bean с квалификатором
  • Некорректное использование @Autowired в конструкторах Spring Bean
  • Внедрение или автоматическое связывание полей/методов в классах, не являющихся валидными Spring Bean

Пример:


public interface FooInterface {...}
  @Component public class FooBean implements FooInterface {...}
  @Component public class OtherBean implements FooInterface {...}
@Component
public class MyComponent {
	@Autowired
	FooInterface foo;  // Невозможно выполнить автоматическое связывание. Существует более одного Bean типа 'FooInterface'.
                     // Beans: fooBean(FooBean.java), otherBean(OtherBean.java)"
}

Пример:


@Component
public class MyComponent {
	@Autowired
	public MyComponent(BarBean bean) {...} // Сообщение "Разрешен только один конструктор с @Autowired"
	@Autowired
	public MyComponent(FooBean bean) {...} // Сообщение "Разрешен только один конструктор с @Autowired"
}
@Component
public class MyFactory {  // Сообщение "Не найден подходящий конструктор с @Autowired"
	public MyFactory(String str) {...}
	public MyFactory(int count) {...}
}

Пример:


public class FooBeanClass {
  @Autowired   // Сообщение 'Автоматическое связывание должно быть определено в валидном Spring Bean: @Component, @Service и т.д.'
  ServiceBean bean;
  }
