Сообщает о методах, возвращаемое значение которых никогда не используется при вызове. Тип возврата таких методов может быть установлен как void.

Не сообщается о методах, аннотированных с помощью @CanIgnoreReturnValue из Error Prone или AssertJ. Быстрое исправление обновляет сигнатуру метода и удаляет операторы return из метода.

Пример:


  // Сообщается при видимости Protected или Public
  protected String myToUpperCase(String s) {
    return s.toUpperCase();
  }
  // Простой сеттер, сообщается в зависимости от настроек
  public String setStr(String str) {
    myStr = str;
    return myStr;
  }
  void test() {
    setStr("value");          // Возвращаемое значение не используется
    myToUpperCase("result");  // Возвращаемое значение не используется
  }

После применения быстрого исправления к обоим методам:


    protected void myToUpperCase(String s) {
      // 'return' полностью удален
      // так как 's.toUpperCase()' не имеет побочных эффектов
    }
    public void setStr(String str) {
      myStr = str;
      // 'return' удален
    }
  ...

Примечание: По соображениям производительности некоторые методы могут не отображаться при подсветке в редакторе. Чтобы увидеть все результаты, запустите проверку через Код | Проверить код или Код | Анализовать код | Запустить проверку по имени

Используйте опцию Игнорировать цепочечные методы, чтобы игнорировать неиспользуемые возвращаемые значения в цепочечных вызовах.

Используйте опцию Максимальная видимость сообщаемых методов, чтобы контролировать максимальную видимость методов, о которых следует сообщать.
