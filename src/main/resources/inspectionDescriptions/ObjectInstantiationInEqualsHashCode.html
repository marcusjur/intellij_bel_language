Сообщает о создании (временных) новых объектов внутри методов equals(), hashCode(), compareTo() и Comparator.compare().

Помимо вызовов конструкторов, новые объекты могут создаваться через автоупаковку или создание итераторов в операторе foreach. Это может привести к проблемам с производительностью, например при добавлении объектов в Set или Map, где эти методы вызываются часто.

Проверка не будет выполняться при создании объектов внутри операторов throw или assert.

Пример:


  class Person {
    private String name;
    private int age;
    public boolean equals(Object o) {
      return Arrays.equals(new Object[] {name, age}, new Object[] {((Foo)o).name, ((Foo)o).age});
    }
    public int hashCode() {
      return (name + age).hashCode();
    }
  }

В этом примере внутри метода equals() создаются два дополнительных массива, использование поля age требует упаковки, а выражение name + age неявно создает новую строку.
