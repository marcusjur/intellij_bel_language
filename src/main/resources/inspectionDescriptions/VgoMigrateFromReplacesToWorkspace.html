Использовать директиву 'replace' для создания файла 'go.work'.

Рабочие пространства Go — это функция, представленная в Go 1.18, которая позволяет работать с несколькими модулями одновременно.
Рабочие пространства являются современной альтернативой директивам 'replace', указывающим на локальные директории.
Все модули в рабочем пространстве имеют одинаковый набор зависимостей, что делает конфигурацию проще и надежнее, а также требует меньше вызовов 'go list' для сканирования структуры проекта.
Исправление этой проверки создаст новый файл 'go.work', но не изменит существующие файлы.

По умолчанию используется уровень серьезности 'warning'. Если созданное рабочее пространство включает модули Go, которые не были заменены в исходном 'go.mod', предупреждения не будут отображаться, так как миграция может иметь побочные эффекты. Тем не менее, вы всё ещё можете запустить исправление вручную.

Подробную информацию о рабочих пространствах смотрите в справочнике по модулям Go.
Для ознакомления с рабочими пространствами Go смотрите Руководство: Начало работы с мультимодульными рабочими пространствами.
