Сообщает об использовании операторов break или continue внутри лямбда-выражений в функциях, похожих на циклы.

Ключевые слова break и continue всегда относятся к фактическим циклам (for, while, do-while). Операторы break и continue никогда не применяются к функциям; например break и continue не работают с forEach, filter, map.

Использование break или continue внутри функций, похожих на циклы (например, forEach), может вызвать путаницу. Эта проверка предлагает добавить метки, чтобы уточнить, к какому оператору относятся Break или Continue.

Поскольку в Kotlin нет понятия функций, похожих на циклы, эта проверка использует эвристический подход. Предполагается, что функции без контракта callsInPlace(EXACTLY_ONCE) или callsInPlace(AT_LEAST_ONCE) являются функциями, похожими на циклы.

Пример:


    for (file in files) {
        file.readLines().forEach { line ->
            if (line == commentMarkerLine) continue
            println(line)
        }
    }

Быстрое исправление добавит поясняющую метку:


    loop@ for (file in files) {
        file.readLines().forEach { line ->
            if (line == commentMarkerLine) continue@loop
            println(line)
        }
    }
