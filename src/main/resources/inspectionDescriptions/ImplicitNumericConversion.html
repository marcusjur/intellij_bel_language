Сообщает о неявных преобразованиях между числовыми типами.

Неявные числовые преобразования сами по себе не являются проблемой, но если они непреднамеренны, могут затруднить отладку ошибок.

Пример:


  double m(int i) {
    return i * 10;
  }

После применения быстрого исправления:


  double m(int i) {
    return (double) (i * 10);
  }

Настройка проверки:

  • Используйте опцию Игнорировать расширяющие преобразования, чтобы игнорировать неявные преобразования, не приводящие к потере данных (например, int->long).
  • Используйте опцию Игнорировать преобразования между 'char' и другими типами, чтобы игнорировать преобразования между char и другими типами. Проверка по-прежнему будет сообщать о преобразованиях между числами с плавающей точкой и другими типами.
  • Используйте опцию Игнорировать преобразования из констант и литералов, чтобы проверка игнорировала преобразования из литералов и констант времени компиляции.
