Сообщает о прямых вызовах panic() или recover() в операторах defer и go.

Такие операторы редко бывают полезны и могут указывать на неправильное использование механизмов panic() и recover(). Особое внимание следует обратить на следующие случаи:

  • go panic(): новая горутина немедленно вызовет панику.
  • defer panic(): функция с таким оператором всегда вызывает панику при завершении.
  • go recovery(): не имеет эффекта, так как новая горутина не может обработать панику.
  • defer recovery(): функция с таким оператором будет молча подавлять панику. Хотя это может работать, идиоматический способ предполагает проверку значения, возвращаемого recover(): 
    defer func() {
  if r := recover(); r != nil {
    fmt.Println("Recovered from: ", r)
  }
}()

Для получения подробной информации об операторах go и обработке паники см. разделы Handling panics и Go statements в спецификации языка Go.
