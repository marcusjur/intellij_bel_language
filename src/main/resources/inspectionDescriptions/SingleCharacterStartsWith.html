Сообщает о вызовах String.startsWith() и String.endsWith(), где в качестве аргумента передаётся строковый литерал из одного символа.

Предлагает быстрое исправление для замены таких вызовов на более эффективную реализацию с использованием String.charAt().

Однако, из-за дополнительной проверки на ненулевую длину, прирост производительности от таких изменений минимален и снижает читаемость кода. Рекомендуется применять это исправление только внутри критичных циклов.

Эта проверка актуальна для Java ME и других сред с сильно ограниченными ресурсами. Необдуманное применение этой проверки может негативно повлиять на ясность и дизайн кода.

Пример:


  boolean startsWithX(String s) {
    return s.startsWith("x");
  }

После применения быстрого исправления:


  boolean startsWithX(String s) {
    return !s.isEmpty() && s.charAt(0) == 'x';
  }
