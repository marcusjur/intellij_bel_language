Сообщать о присваиваниях, результат которых не используется. Если присваивание не используется, лучше его удалить для сокращения кода и избежания избыточных операций.

Будут отмечены следующие случаи:

  • Переменная никогда не читается после присваивания
  • Значение переменной перезаписывается до того, как было прочитано
  • Избыточный инициализатор переменной (по одной из двух причин выше)

Настройка проверки:

Используйте опцию Сообщать об избыточных инициализаторах для обнаружения избыточной инициализации: 


  int getI() {
    int i = 0; // Избыточная инициализация
    i = 2;
    return i;
  }

Используйте опцию Сообщать о префиксных выражениях, которые можно заменить бинарными для обнаружения случаев, где выражение ++i можно заменить на i + 1: 


  int preInc(int value) {
    int res = value;
    return ++res;
  }

Используйте опцию Сообщать о постфиксных выражениях, где измененное значение не используется для обнаружения случаев i++, где значение i далее не используется: 


  int postInc(int value) {
    int res = value;
    return res++;
  }

Используйте опцию Сообщать о переменных шаблона, значение которых никогда не используется для обнаружения случаев, когда значение переменной шаблона перезаписывается до чтения: 


  if (object instanceof String s) {
      s = "hello";
      System.out.println(s);
  }

Используйте опцию Сообщать о неиспользуемых параметрах итерации для обнаружения случаев, когда значение итерационного параметра в расширенном операторе for перезаписывается до чтения: 


  for (String arg : args) {
      arg = "test";
      System.out.println(arg);
  }
