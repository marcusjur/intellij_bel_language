Сообщает об операторах if, while, do или for без фигурных скобок. Некоторые стили кодования, такие как Google Java Style Guide, требуют, чтобы все управляющие операторы имели фигурные скобки.

При добавлении дополнительных операторов к управляющей конструкции без фигурных скобок важно не забыть добавить фигурные скобки. Также нужно быть особенно осторожным при комментировании строки кода без использования фигурных скобок, чтобы случайно не сделать следующий оператор частью управляющей конструкции. Постоянное использование фигурных скобок делает добавление или комментирование строк кода более безопасным.

Если всегда следовать стилю кодования с использованием фигурных скобок, то уязвимость типа goto fail вряд ли возникнет. Управляющие конструкции с фигурными скобками делают поток управления более очевидным, не полагаясь на отступы, которые могут быть неправильными.

Пример:


  class Strange {
    void x(boolean one, boolean two) {
      if(one)
        if(two)
          foo();
      else
        bar();
    }
    void foo() {}
    void bar() {}
  }

Быстрое исправление с обёрткой тела операторов в фигурные скобки:


  class Strange {
    void x(boolean one, boolean two) {
      if(one) {
        if(two) {
          foo();
        } else {
          bar();
        }
      }
    }
    void foo() {}
    void bar() {}
  }
