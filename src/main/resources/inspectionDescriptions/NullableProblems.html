Сообщает о проблемах, связанных с аннотациями на null.

Примеры:

  • Переопределённый метод без аннотации: 
    
abstract class A {
  @NotNull abstract String m();
}
class B extends A {
    String m() { return "empty string"; }
}
  • Аннотированный примитивный тип: @NotNull int myFoo;
  • @Nullable и @NotNull на одном элементе: @Nullable @NotNull String myFooString;
  • Коллекция с nullable элементами присваивается коллекции с non-null элементами: 
    
void testList(List<@Nullable String> nullableList) {
   List<@NotNull String> list2 = nullableList;
}

Используйте кнопку Настроить аннотации для указания аннотаций на null, и флажки для точной настройки мест, где проверка будет выдавать предупреждения.

Проверка выполняется только если уровень языка проекта или модуля 5 или выше, и аннотации на null доступны в classpath.
