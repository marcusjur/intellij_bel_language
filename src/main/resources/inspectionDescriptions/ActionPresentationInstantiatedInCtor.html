Сообщает о любых действиях, зарегистрированных в файле plugin.xml, и создает экземпляр объекта com.intellij.openapi.actionSystem.Presentation в своем конструкторе.

Любой конструктор AnAction с параметрами создает экземпляр объекта Presentation. Однако создание экземпляра объекта Presentation в конструкторе приводит к выделению ресурсов, что может быть необязательным. Вместо создания экземпляра Presentation для хранения текста, описания или значков, более эффективно использовать конструктор без параметров AnAction и других базовых классов, следуя соглашению об установке текста, описания и значков в plugin.xml. ИСР загружает текст, описание и значки только тогда, когда действие фактически отображается в пользовательском интерфейсе.

Соглашения по установке текста, описания и значков следующие:

  • Установите атрибут id для действия в файле plugin.xml.
  • Если нужен значок, также установите атрибут icon.
  • Установите текст и описание в соответствующем пакете сообщений (может быть переопределено в <actions>):
    • action.<action-id>.text=Переведенный текст действия
    • action.<action-id>.description=Переведенное описание действия

Неправильный пример:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction(
    KotlinBundle.message("action.new.file.text"),
    KotlinBundle.message("action.new.file.description"),
    KotlinIcons.FILE
  )


  <action
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
  </action>

Правильный пример:


  // NewKotlinFileAction.kt
  internal class NewKotlinFileAction : AnAction()


  <action
    id="Kotlin.NewFile"
    class="org.jetbrains.kotlin.idea.actions.NewKotlinFileAction"
    icon="org.jetbrains.kotlin.idea.KotlinIcons.FILE">
  </action>


  # KotlinBundle.properties
  action.Kotlin.NewFile.text=Класс/файл Kotlin
  action.Kotlin.NewFile.description=Создать новый класс или файл Kotlin
