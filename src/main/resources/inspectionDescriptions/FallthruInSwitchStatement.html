Сообщает о сквозном выполнении в операторе switch.

Сквозное выполнение происходит, когда последовательность исполняемых операторов после метки case не гарантирует передачу управления перед следующей меткой case. Это может произойти, например, когда в ветке отсутствует оператор break. В таких случаях управление переходит к операторам после метки switch, даже если выражение switch не равно значению метки, к которой произошло сквозное выполнение. Хотя иногда это делается намеренно, такая структура может вызывать путаницу и обычно является результатом опечатки.

Эта проверка будет игнорировать любое сквозное выполнение, которое сопровождается текстовым комментарием, соответствующим регулярному выражению (?i)falls?\s*thro?u.

Исправить можно, добавив оператор break в ветки, где возможно сквозное выполнение к следующей ветке.

Пример:


    switch(x) {
      case (4):
          if (condition) {
              System.out.println("3");
              // Здесь нет break
          } else {
              break;
          }
      case (6):
          System.out.println("4");
    }

После применения быстрого исправления:


    switch(x) {
        case (4):
            if (condition) {
                System.out.println("3");
            } else {
                break;
            }
            break;
        case (6):
            System.out.println("4");
    }
