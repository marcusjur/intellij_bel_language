Сообщение о длинных цепочках преобразований Reactive Streams.

Каждый вызов метода оператора, например map() или filter(), создает определённые объекты для этих операторов. Длинная цепочка операторов, вызываемая для каждого элемента потока при каждой подписке, может привести к снижению производительности. Чтобы избежать этого, по возможности объединяйте длинные цепочки операторов в один вызов оператора.

Пример:


Flux.just(1, 2, 3)
        .map(it -> it + 1)
        .map(it -> it + 2)
        .map(it -> it + 3)

После применения быстрого исправления:


Flux.just(1, 2, 3)
        .map(it -> it + 1 + 2 + 3)
