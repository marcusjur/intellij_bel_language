Сообщает о методах, которые не являются static, private, final или abstract, и имеют непустое тело.

Программование с использованием стиля, избегающего таких методов, защищает контракты класса от нарушения их подклассами. Преимущество такого стиля в том, что подклассы не могут нарушить состояние суперкласса из-за забытого вызова super-метода. Недостатком является ограниченная гибкость подклассов, особенно в том, что они не могут предотвратить выполнение кода в суперклассе. Быстрое исправление может добавить отсутствующие модификаторы.

Пример:


  class Foo {
    public boolean equals(Object o) { return true; }
  }

После применения быстрого исправления:


  class Foo {
    public final boolean equals(Object o) { return true; }
  }
Эта проверка подходит для кода, который будет использоваться в защищенной среде, и может быть неуместна в менее ограниченных средах.
