Сообщает об исключениях, которые объявлены в сигнатуре метода, но никогда не выбрасываются ни самим методом, ни его реализациями или переопределениями.

Эта проверка игнорирует методы, связанные с сериализацией, такие как readObject() и writeObject().

Пример:


  void method() throws InterruptedException {
    System.out.println();
  }

Быстрое исправление удалит ненужные исключения из объявления и нормализует избыточные конструкции try-catch:


  void method() {
    System.out.println();
  }

Примечание: По соображениям производительности некоторые исключения могут не отображаться при подсветке в редакторе. Чтобы увидеть все результаты, запустите проверку, выбрав Код | Проверить код или Код | Анализовать код | Запустить проверку по имени в главном меню.

Используйте опцию Игнорировать исключения, выбрасываемые методами точки входа, чтобы не получать сообщения об исключениях, выбрасываемых методами вроде main(). Методы точки входа можно настроить в параметрах проверки Java | Избыточные объявления | Неиспользуемые объявления.
