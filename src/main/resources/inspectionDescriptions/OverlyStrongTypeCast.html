Сообщает о слишком строгом приведении типов. Например, когда объект приводится к типу ArrayList, хотя достаточно было бы привести его к List.

Примечание: Эта проверка очень похожа на проверку избыточного приведения типов. Если вы намеренно используете более строгое приведение для генерации ClassCastException, применение исправления для этой проверки может изменить семантику программы.

Пример:


  interface Super {
    void doSmth();
  }
  interface Sub extends Super { }
  void use(Object obj) {
    // Предупреждение: можно использовать ((Super)obj).doSmth()
    ((Sub)obj).doSmth();
  }
