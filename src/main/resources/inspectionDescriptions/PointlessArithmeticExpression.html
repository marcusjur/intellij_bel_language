Сообщает о бессмысленных арифметических выражениях. Такие выражения включают прибавление 0, вычитание 0, умножение на 0 или 1, а также деление на 1.

Такие выражения могут появиться в результате автоматического рефакторинга и вряд ли являются намеренными.

Быстрое исправление упростит такие выражения.

Пример: 


  void f(int a) {
    int x = a - a;
    int y = a + 0;
    int res = x / x;
  }

После применения быстрого исправления:


  void f(int a) {
    int x = 0;
    int y = a;
    int res = 1;
  }

Обратите внимание, что в редких случаях предложенная замена может быть не полностью эквивалентна исходному коду для всех возможных входных данных. Например, проверка предложит заменить x / x на 1. Однако если x равен нулю, исходный код выбросит ArithmeticException или приведет к NaN. Кроме того, если x равен NaN, результат также будет NaN. Такое поведение вряд ли является намеренным.
