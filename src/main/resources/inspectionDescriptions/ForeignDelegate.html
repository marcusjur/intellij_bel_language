Обнаружение потенциально неправильно размещенных вызовов методов Gradle.

Скрипты сборки Gradle содержат множество вложенных замыканий, что делает структуру кода похожей на язык разметки. Из-за особенностей работы DSL методы, которые можно написать во внешнем замыкании, также доступны во внутреннем замыкании. Такие методы могут не иметь смысла при написании вне своей области видимости.
Иногда такие случаи сложно обнаружить. Цель этой проверки — найти подобные методы.

Пример:


repositories {
  // делегат 'repositories' имеет метод 'exclusiveContent', который можно использовать здесь
  maven {
    // делегат 'maven' имеет метод 'content', который можно использовать здесь
    // но также можно использовать 'exclusiveContent'
    exclusiveContent {} // сообщает об 'exclusiveContent'
  }
}
