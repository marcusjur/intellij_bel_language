Сообщать о выражениях условий с отрицанием.

Изменение порядка ветвей условного выражения часто повышает читаемость таких конструкций.

Используйте опции Игнорировать сравнение '!= null' и Игнорировать сравнение '!= 0', чтобы исключить проверки вида obj != null или num != 0. Поскольку obj != null фактически означает "obj существует", смысл всего выражения не содержит отрицания и легко понимается.

Такая же логика применима к выражениям вида num != 0, особенно при работе с битовыми масками.

Дополнительное преимущество такого подхода в том, что значимый случай рассматривается первым. В большинстве случаев ветвь с == null возвращает сам null, как показано в примере ниже: 


    static String getName(Person p) {
        return p != null ? p.getName() : null;
    }
    static String getExecutableString(int fileMode) {
        return (fileMode & 0b001001001) != 0 ? "executable" : "non-executable";
    }
