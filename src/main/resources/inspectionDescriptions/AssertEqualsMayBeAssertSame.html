Обнаружены вызовы JUnit assertEquals(), которые могут быть заменены эквивалентными вызовами assertSame(). Это может происходить, когда аргументы являются экземплярами final класса, который не переопределяет метод Object.equals() и явно сравнивает идентичность объектов.

Рекомендуется заменить assertEquals() на assertSame().

Пример:


  @Test
  public void testObjectType() {
    Object o = getObject();
    Assert.assertEquals(String.class, o.getClass());
  }

После применения быстрого исправления:


  @Test
  public void testSort() {
    Object o = getObject();
    Assert.assertSame(String.class, o.getClass());
  }
