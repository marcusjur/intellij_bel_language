Обнаружение пустых классов и пустых Java-файлов.

Класс считается пустым, если он не содержит полей, методов, конструкторов или инициализаторов. Пустые классы иногда остаются после значительных изменений или рефакторинга.

Пример:


  class Example {
    List<String> getList() {
       return new ArrayList<>() {
       };
    }
  }

После применения быстрого исправления:


  class Example {
    List<String> getList() {
       return new ArrayList<>();
    }
  }

Настройка проверки:

  • Используйте опцию Игнорировать при наличии аннотаций, чтобы указать специальные аннотации. Проверка будет игнорировать классы, помеченные этими аннотациями.

  • Используйте опцию Игнорировать, если класс параметризован родительским типом, чтобы игнорировать классы с параметризованным суперклассом. Например: 

    class MyList extends ArrayList<String> {}
  • Используйте опцию Игнорировать подклассы java.lang.Throwable, чтобы игнорировать классы, расширяющие java.lang.Throwable.
  • Используйте опцию Считать комментарии содержимым, чтобы игнорировать классы, содержащие комментарии.
