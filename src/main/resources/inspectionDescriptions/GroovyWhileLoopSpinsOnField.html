Сообщает о циклах while, которые выполняют спин на не-volatile поле, ожидая его изменения другим потоком.

Такие циклы не только могут потреблять много процессорного времени, если внутри цикла выполняется минимальная работа, но и, вероятно, имеют семантику, отличную от ожидаемой. Java Memory Model допускает, что такой цикл может никогда не завершиться, даже если другой поток изменит значение поля.

Пример:


  class SpinsOnField {
      boolean ready = false;
      void run() {
          // Этот цикл может никогда не завершиться,
          // даже после вызова markAsReady из другого потока
          while (!ready) {
          }
          // Выполнение некоторой работы
      }
      void markAsReady() {
          ready = true;
      }
  }

Кроме того, начиная с Java 9, рекомендуется вызывать Thread.onSpinWait() в циклах спина на volatile полях, что может значительно улучшить производительность на определённом оборудовании.
