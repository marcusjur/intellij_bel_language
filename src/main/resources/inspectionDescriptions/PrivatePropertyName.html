Сообщает о именах private-свойств, которые не соответствуют рекомендуемым соглашениям об именовании.

Согласованное именование упрощает чтение и понимание кода. Согласно официальному руководству по стилю Kotlin, имена private-свойств должны начинаться с маленькой буквы и использовать camelCase. Также допускается использование префикса с подчеркиванием, но только для private свойств.

Дополнительные правила именования можно ввести, изменив регулярное выражение "Pattern".

Пример:


  val _My_Cool_Property = ""

Быстрое исправление переименует в соответствии с соглашениями об именовании Kotlin:


  val _myCoolProperty = ""
