Сообщать о методах с модификатором synchronized.

Использование модификатора synchronized для методов может быть нежелательным по нескольким причинам:

  1. При блокировке следует выполнять как можно меньше операций. Поэтому обычно лучше использовать блок synchronized, ограничивая только код, работающий с общим состоянием.
  2. Синхронизация становится частью интерфейса метода. Это затрудняет переход к другим механизмам блокировки.
  3. Становится сложнее отслеживать, что именно блокируется для конкретного объекта.
  4. При наследовании методов становятся возможными как преднамеренные, так и непреднамеренные DoS-атаки (отказ в обслуживании).

Альтернативным решением является синхронизация по private final объекту блокировки, доступ к которому полностью контролируется.

Быстрое исправление может обернуть тело метода в блок synchronized(this).

Пример:


  class Main {
    public synchronized void fooBar() {
    }
  }

После применения быстрого исправления:


  class Main {
    public void fooBar() {
      synchronized (this) {
      }
    }
  }

Для этой проверки можно настроить следующие параметры:

  1. Включать нативные методы - включить нативные методы в область проверки.
  2. Игнорировать методы, переопределяющие synchronized методы - не сообщать о методах, которые переопределяют методы с модификатором synchronized.
