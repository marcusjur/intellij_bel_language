Сообщает о классах, реализующих Serializable, но не объявляющих поле serialVersionUID.

Без поля serialVersionUID любые изменения в классе приведут к нечитаемости ранее сериализованных версий.

Пример:


  class Main implements Serializable {
  }

После применения быстрого исправления:


  class Main implements Serializable {
      private static final long serialVersionUID = -1446398935944895849L;
  }
При использовании уровня языка JDK 14 или выше быстрое исправление также добавит аннотацию java.io.Serial.

Настройте проверку с помощью следующих параметров:

  • Укажите классы, наследники которых не должны проверяться. Это применимо к классам, которые наследуют Serializable от суперкласса, но не используются для сериализации.
  • Следует ли игнорировать анонимные классы, реализующие Serializable.
