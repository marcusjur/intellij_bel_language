Выражение может быть заменено вызовом метода Integer.compare() или аналогичных методов из классов Long, Short, Byte, Double или Float (вместо более многословных или менее эффективных конструкций).

Если x и y являются упакованными целыми числами, рекомендуется использовать x.compareTo(y); если они являются примитивами, рекомендуется использовать Integer.compare(x, y).

Пример:


  public int compare(int x, int y) {
    return x > y ? 1 : x < y ? -1 : 0;
  }

После применения быстрого исправления:


  public int compare(int x, int y) {
    return Integer.compare(x, y);
  }

Обратите внимание, что Double.compare и Float.compare немного изменяют семантику кода. В частности, они позволяют различать -0.0 и 0.0 (Double.compare(-0.0, 0.0) возвращает -1). Кроме того, они последовательно обрабатывают значения NaN. В большинстве случаев такое изменение семантики фактически улучшает код. Если в вашем случае изменение семантики неприемлемо, отключите проверку для чисел с плавающей точкой с помощью флажка.
