Сообщает о следующих ошибках в запросах Persistence QL:
  • Несоответствие типов выражений
  • Некорректные параметры
  • Пустые или константные условия
  • Неразрешённые символы

Примеры запросов:


SELECT OBJECT(e) FROM JavaEntity e WHERE TRUE // Предупреждение: константное условие
SELECT OBJECT(e) FROM JavaEntity e WHERE e.id BETWEEN 0 AND '2' // Ошибка: ожидалось числовое значение, получена строка
SELECT e from JavaEntity e where e.id=?1 and e.name=?3 // Ошибка: только 2 фактических параметра запроса
