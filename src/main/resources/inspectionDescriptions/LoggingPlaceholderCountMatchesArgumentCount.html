Сообщает о вызовах логования SLF4J, Log4j2 и akka.event.LoggingAdapter, например logger.info("{}: {}", key), где количество заполнителей {} в сообщении логгера не соответствует количеству дополнительных аргументов в вызове логования.

Эта опция проверки позволяет указать, какая реализация используется для SLF4J. Если выбрана опция Автоопределение, выполняется поиск org.apache.logging.slf4j.Log4jLogger в classpath. Если файл найден или выбрана опция Да, случаи с заполнителем для последнего параметра типа исключения не будут отмечаться для SLFJ4 API.
Например: 


//этот случай не будет отмечен, если выбрана опция "Да"
log.error("For id {}: {}", "1", new RuntimeException());
В этом случае 'new RuntimeException()' будет выведен с помощью 'toString()' (без вывода стека вызовов): 

For id 1: java.lang.RuntimeException
В противном случае это будет подсвечено, так как последний заполнитель не используется: 

For id 1: {}
java.lang.RuntimeException: null
Опция Нет может использоваться для подсветки таких случаев всегда, когда заполнитель используется для исключения, даже если org.apache.logging.slf4j.Log4jLogger используется в качестве бэкенда.
Эта опция применяется только к SLF4J.
