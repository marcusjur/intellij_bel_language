Сообщает о полях transient, которые инициализуются в процессе обычного конструирования объекта, но их класс не имеет метода readObject.

Поля transient не сериализуются, поэтому во время десериализации их необходимо инициализовать отдельно в методе readObject().

Любые поля transient, которые не инициализуются в процессе обычного конструирования объекта, считаются использующими инициализацию по умолчанию и не отмечаются данной проверкой.

Пример:


  class Person implements Serializable {
    transient String name = "Default"; //предупреждение: после десериализации может быть null
    transient String surname; //null считается значением по умолчанию, не отмечается
  }
