Сообщается, что оператор until может быть заменен на оператор ..<.

Каждая замена until на ..< не меняет семантику кода.

Исследования пользовательского опыта показывают, что разработчики допускают примерно на 20-30% меньше ошибок при чтении кода с ..< по сравнению с until.

Пример:


  fun main(args: Array<String>) {
      for (index in 0 until args.size) {
          println(index)
      }
  }

После применения быстрого исправления:


  fun main(args: Array<String>) {
      for (index in 0..<args.size) {
          println(index)
      }
  }

Эта проверка выполняется только если уровень языка Kotlin в проекте или модуле 1.7 или выше.
