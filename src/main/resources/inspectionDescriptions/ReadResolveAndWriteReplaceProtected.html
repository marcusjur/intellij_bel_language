Сообщает о классах, реализующих java.io.Serializable, в которых методы readResolve() или writeReplace() не объявлены как protected.

Объявление методов readResolve() и writeReplace() как private может привести к их неявному игнорированию подклассами, а объявление их как public позволяет вызывать их из недоверенного кода.

Если содержащий класс объявлен как final, эти методы могут быть объявлены как private.

Пример:


  class ClassWithSerialization implements Serializable {
    public Object writeReplace() { // Предупреждение: 'writeReplace()' не объявлен как protected
        ...
    }
  }
