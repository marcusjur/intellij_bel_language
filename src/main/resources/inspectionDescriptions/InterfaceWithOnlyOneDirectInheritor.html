Обнаружен интерфейс, имеющий только одного прямого наследника. Хотя такие интерфейсы могут обеспечивать хорошую ясность дизайна в средах с ограниченной памятью или пропускной способностью, они излишне увеличивают общий объем приложения. Рекомендуется рассмотреть возможность объединения интерфейса с его наследником.

Эта проверка актуальна для Java ME и других сред с сильно ограниченными ресурсами. Бездумное применение этой проверки может негативно повлиять на ясность кода и дизайн.
