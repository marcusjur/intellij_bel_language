Сообщает о вызовах следующих методов объекта java.util.Properties:
  • put()
  • putIfAbsent()
  • putAll()
  • get()

По историческим причинам java.util.Properties наследуется от java.util.Hashtable, но использование этих методов не рекомендуется во избежание загрязнения значений свойств типами, отличными от String.

Вызовы java.util.Properties.putAll() не отмечаются, когда ключи и значения в отображении имеют тип String. Такие вызовы безопасны, и лучшей альтернативы нет.

Пример:


  Object f(Properties props) {
    props.put("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.get("Hello");
  }

После применения быстрого исправления:


  Object f(Properties props) {
    props.setProperty("hello", "world");
    props.putIfAbsent("hello", "world");
    props.putAll(new HashMap<>());
    return props.getProperty("hello");
  }
