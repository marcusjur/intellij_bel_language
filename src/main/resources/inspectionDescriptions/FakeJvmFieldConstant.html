Сообщение о свойстве Kotlin, которое не является const и используется как аргумент Java-аннотации.

Например, свойство с аннотацией @JvmField, имеющее инициализатор, который может быть вычислен во время компиляции, и имеющее примитивный тип или тип String.

Такие свойства в байт-коде Kotlin 1.1-1.2 имеют атрибут ConstantValue. Этот атрибут позволяет javac выполнять встраивание использования соответствующего поля и использовать его в аннотациях. Это может привести к некорректному поведению при раздельной или инкрементальной компиляции смешанного Java/Kotlin кода. Такое поведение может измениться в Kotlin 1.3 (атрибут ConstantValue больше не будет добавляться).

Пример:

Код Kotlin в файле foo.kt:


  annotation class Ann(val s: String)
  @JvmField val importantString = "important"

Код Java:


  public class JavaUser {
      // Это опасно
      @Ann(s = FooKt.importantString)
      public void foo() {}
  }

Чтобы исправить эту проблему, замените аннотацию @JvmField на модификатор const для соответствующего свойства Kotlin или выполните встраивание значения.
