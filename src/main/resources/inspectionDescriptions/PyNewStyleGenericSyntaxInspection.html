Обнаружено недопустимое использование синтаксиса параметров типа согласно PEP 695

Проверьте следующие проблемы в определениях функций и классов, а также в объявлениях псевдонимов типов нового стиля:

  • Расширение typing.Generic в обобщенных классах нового стиля
  • Расширение параметризованного typing.Protocol в обобщенных классах нового стиля
  • Использование обобщенных ограничений и границ с параметрами типа для ParamSpec и TypeVarTuple
  • Смешивание переменных типа традиционного и нового стиля
  • Использование традиционных переменных типа в псевдонимах типов нового стиля

Примеры: 


  from typing import Generic
  class Example[T](Generic[T]): ... # Класс с списком параметров типа не должен расширять 'Generic'


  class Example[T: (list[S], str)]: ... # Обобщенные типы не допускаются в ограничениях и границах параметров типа


  from typing import TypeVar
  K = TypeVar("K")
  class ClassC[V]:
      def method2[M](self, a: M, b: K) -> M | K: ... # Смешивание TypeVars традиционного и нового стиля не допускается
