Сообщение о достижимости runBlocking построителя из корутины.

Корутины разделяют поток выполнения. Когда построитель runBlocking вызывается из корутины, он блокирует базовый поток и препятствует использованию этого ресурса другими корутинами. Это может привести к проблемам с производительностью, а в худшем случае - к истощению потоков.

Предоставляется потенциальный стек вызовов от корутинных примитивов (функций приостановки или построителей) до построителя runBlocking.

Пример:


  suspend fun main() {
      foo() 
   }
  fun foo() {
      runBlocking { suspendFunction() }
  }

Возможное решение:


  suspend fun main() {
      foo() 
   }
  suspend fun foo() {
      suspendFunction()
  }

Используйте опцию Просмотр функций с переопределениями для настройки анализа виртуальных функций:

  • Используйте опцию Нет, чтобы пропустить анализ функций с переопределениями.
  • Используйте опцию Да, исключая переопределения, чтобы просматривать только функции в определённом типе вызываемого объекта.
  • Используйте опцию Да, включая переопределения, чтобы просматривать функции и все их переопределения.
