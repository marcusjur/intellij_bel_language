Сообщает о коде, который использует == или != вместо equals() для проверки равенства объектов.

Использование операторов == или != для сравнения объектов обычно является ошибкой, так как они проверяют идентичность объектов, а не их равенство. Сравнения с null не отмечаются.

Сравнения массивов, String и Number отмечаются отдельными проверками.

Пример:

  if (list1 == list2) {
    return;
  }

После применения быстрого исправления:

  if (Objects.equals(list1, list2)) {
    return;
  }

Используйте настройки проверки для конфигурации исключений для данной проверки.
