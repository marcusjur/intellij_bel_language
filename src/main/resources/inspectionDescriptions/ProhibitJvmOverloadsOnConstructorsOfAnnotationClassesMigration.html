Сообщение о наличии @JvmOverloads в конструкторе аннотации, так как это не имеет смысла.

@JvmOverloads не занимает места в сгенерированном байт-коде и метаданных Kotlin, поэтому не влияет на сгенерированный байт-код и поведение программы.

Начиная с Kotlin 1.4, использование @JvmOverloads в конструкторе класса аннотации вызывает ошибку компиляции.

Пример:


  annotation class A @JvmOverloads constructor(val x: Int = 1)

После применения быстрого исправления:


  annotation class A constructor(val x: Int = 1)
