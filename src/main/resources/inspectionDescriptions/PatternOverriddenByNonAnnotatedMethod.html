Сообщает о случаях, когда метод без аннотации @Pattern переопределяет метод с аннотацией @Pattern. Это не препятствует подсветке ошибок в редакторе, но переопределённый метод не будет проверяться во время выполнения.

Доступно быстрое исправление для добавления аннотации @Pattern, соответствующей аннотации в методе суперкласса. Это обеспечивает корректную работу проверок во время выполнения.

Пример:


  abstract class Parent {
    abstract @Pattern("\\d\\d-\\d\\d\\d") String getId();
  }
  class Child extends Parent {
    @Override String getId() { // предупреждение здесь
      return "12-345";
    }
  }
