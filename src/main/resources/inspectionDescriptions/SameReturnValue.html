Сообщает о методах и иерархиях методов, которые всегда возвращают одну и ту же константу.

Эта проверка работает по-разному в пакетном режиме (с помощью Код | Проверить код или Код | Анализовать код | Запустить проверку по имени) и при динамическом выполнении в редакторе:

  • В пакетном режиме проверка сообщает о методах и иерархиях методов, которые всегда возвращают одну и ту же константу.
  • В редакторе проверка сообщает только о методах с несколькими операторами return, без super-методов и методах, которые не могут быть переопределены. Если метод переопределяет или реализует другой метод, соглашение может требовать возврата определённой константы, но при этом мы можем хотеть иметь несколько точек выхода. Если метод может быть переопределен, он может возвращать различные значения в подклассах.

Пример:


  class X {
                // Предупреждение только в пакетном режиме:
    int xxx() { // Метод 'xxx()' и все его переопределения всегда возвращают '0'
      return 0;
    }
  }
  class Y extends X {
    @Override
    int xxx() {
        return 0;
    }
                // Предупреждение только в пакетном режиме:
    int yyy() { // Метод 'yyy()' всегда возвращает '0'
        return 0;
    }
                                  // Предупреждение как в пакетном режиме, так и при динамическом выполнении:
    final int zzz(boolean flag) { // Метод 'zzz()' всегда возвращает '0'
        if (Math.random() > 0.5) {
            return 0;
        }
        return 0;
    }
  }
