Сообщает о неисчерпывающих выражениях when, которые будут вызывать ошибки компиляции начиная с версии 1.7.

Типы мотивации:

  • Проблематичные/бессмысленные шаблоны использования, которые следует предотвращать (например, неинтуитивное поведение)
    • Код, подверженный ошибкам
  • Несоответствия в дизайне (различная обработка в разных контекстах)

Типы воздействия:

  • Компиляция. Некоторый код, который раньше компилировался, перестанет компилироваться
    • В некоторых случаях такой код может работать без исключений
      • Часть такого кода может компилироваться без каких-либо предупреждений

Дополнительная информация: KT-47709: Сделать выражения when с перечислениями, sealed классами и логическими значениями исчерпывающими по умолчанию

Быстрое исправление добавит отсутствующую ветвь else -> {}.

Пример:


  sealed class Base {
      class A : Base()
      class B : Base()
  }
  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
      }
  }

После применения быстрого исправления:


  sealed class Base {
      class A : Base()
      class B : Base()
  }
  fun test(base: Base) {
      when (base) {
          is Base.A -> ""
          else -> {}
      }
  }

Эта проверка выполняется только если уровень языка Kotlin для проекта или модуля установлен на 1.6 или выше.
