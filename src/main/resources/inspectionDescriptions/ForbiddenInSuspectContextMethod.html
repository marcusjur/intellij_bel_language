Сообщает о неправильном использовании методов в корутинах Kotlin, использующих контекст потока (с аннотацией @RequiresBlockingContext). Для многих из этих методов существуют корутин-совместимые альтернативы, которые можно использовать в контексте suspend. Примеры:
  • com.intellij.openapi.progress.ProgressManager.checkCanceled() следует заменить на com.intellij.openapi.progress.checkCancelled()
  • com.intellij.openapi.application.Application.invokeAndWait() следует заменить на withContext(Dispatchers.EDT)
  • и т.д.
