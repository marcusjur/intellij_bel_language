Сообщение о доступе к нефинальным полям в реализации метода compareTo().

Такой доступ может привести к тому, что метод compareTo() будет возвращать разные результаты в разные моменты жизненного цикла объекта, что может вызвать проблемы при использовании стандартных коллекций, таких как java.util.TreeSet.

Быстрое исправление для установки поля как final доступно только в том случае, если отсутствует доступ на запись к этому полю. В противном случае исправление не рекомендуется.

Пример:


  class Foo implements Comparable<Foo>{
    private int index;
    Foo(int idx) {
      index = idx;
    }
    @Override
    public int compareTo(Foo foo) {
     return Integer.compare(this.index, foo.index);
    }
  }

После применения быстрого исправления:


  class Foo implements Comparable<Foo>{
    private final int index;
    Foo(int idx) {
      index = idx;
    }
    @Override
    public int compareTo(Foo foo) {
      return Integer.compare(this.index, foo.index);
    }
  }
