Обнаружение необязательных аннотаций opt-in, которые можно безопасно удалить.

Код, использующий экспериментальные API, требует аннотации @OptIn, поскольку такие API могут измениться в будущем. Если такие API больше не используются (например, когда экспериментальный API стал стабильным и больше не требует явного подключения), эта аннотация становится избыточной и может вводить в заблуждение.

Быстрое исправление Удалить аннотацию позволяет удалить ненужные аннотации @OptIn.

Пример:


  @OptIn(ExperimentalApi::class)
  fun foo(x: Bar) {
      x.baz()
  }

После применения быстрого исправления:


  fun foo(x: Bar) {
      x.baz()
  }
