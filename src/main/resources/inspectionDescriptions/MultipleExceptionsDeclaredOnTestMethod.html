Сообщает о методах тестования JUnit с множественными исключениями в блоке throws. Такие объявления создают ненужное многословие. Тестовые методы не будут вызываться из другого проектного кода, поэтому нет необходимости обрабатывать эти исключения по отдельности.

Пример:


  @Test
  public void testReflection() throws NoSuchMethodException,
      InvocationTargetException, IllegalAccessException {
    String result = (String) String.class.getMethod("trim")
        .invoke(" hello ");
    assertEquals("hello", result);
  }

С помощью быстрого исправления замените объявления исключений на единое исключение:


  @Test
  public void testReflection() throws Exception {
    String result = (String) String.class.getMethod("trim")
        .invoke(" hello ");
    assertEquals("hello", result);
  }
