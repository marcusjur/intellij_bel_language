Обнаружены нарушения закона Деметры.

Закон Деметры не является строгим правилом, но он определяет стилистическое руководство: никогда не вызывайте методы объектов, полученных из другого вызова. Код, следующий этому принципу, легче поддерживать, адаптировать и рефакторить, он имеет меньшую связанность между методами, меньше дублирования и лучшее сокрытие информации. С другой стороны, для соблюдения этого принципа может потребоваться написание множества методов-оберток.

Пример:


  boolean pay(Customer c, Invoice invoice) {
    int dollars = c.getWallet().contents; // нарушение
    if (dollars >= invoice.getAmount()) {
      Wallet w = c.getWallet();
      w.subtract(invoice.getAmount()); // нарушение
      return true;
    }
    return false;
  }
Приведенный выше пример лучше реализовать как метод payInvoice(Invoice invoice) в классе Customer.

Используйте опцию игнорировать вызовы методов библиотек и доступ к полям библиотек, чтобы исключить нарушения закона Деметры, которые невозможно исправить без изменения библиотек.
