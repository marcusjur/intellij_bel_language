Сообщает об объектах классов, не реализующих java.io.Serializable, которые используются в качестве аргументов в методах javax.servlet.http.HttpSession.setAttribute() или javax.servlet.http.HttpSession.putValue().

Если происходит пассивация или миграция HttpSession, такие объекты не будут сериализованы, что может привести к трудно диагностируемым ошибкам.

Эта проверка предполагает, что объекты типов java.util.Collection и java.util.Map являются Serializable, если только их параметры типа не являются не-Serializable.

Пример:


  void foo(HttpSession session) {
      session.setAttribute("foo", new NonSerializable());
  }
  static class NonSerializable {}
