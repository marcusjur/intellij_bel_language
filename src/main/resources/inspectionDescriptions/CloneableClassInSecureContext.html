Отчёт о классах, которые могут быть клонованы.

Класс может быть клонован, если он поддерживает интерфейс Cloneable и его метод clone() не определен как немедленно выбрасывающий исключение. В коде, используемом для обеспечения безопасности, клонируемые классы могут представлять опасность.

Пример:

class SecureBean implements Cloneable {}

После применения быстрого исправления:

class SecureBean {}

Когда класс расширяет существующий клонируемый класс или реализует интерфейс клонования, после применения быстрого исправления код может выглядеть следующим образом: 

class SecureBean extends ParentBean {
    @Override
    protected SecureBean clone() throws CloneNotSupportedException {
        throw new CloneNotSupportedException();
    }
}
