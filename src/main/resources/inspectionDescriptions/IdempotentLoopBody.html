Сообщает о циклах, которые создают побочные эффекты только в первой итерации, но не создают их во второй и последующих итерациях, что может указывать на ошибку программования.

Такие циклы могут выполняться ноль раз, один раз или бесконечно. Если нет вероятности бесконечного цикла, его можно заменить оператором if. В противном случае программа может зависнуть.

Пример:


    public void foo(String baseName, String names) {
        int suffix = 1;
        String name = baseName;
        while (names.contains(name)) {
            // Ошибка: suffix не обновляется, что делает тело цикла идемпотентным
            name = baseName + suffix;
        }
    }
