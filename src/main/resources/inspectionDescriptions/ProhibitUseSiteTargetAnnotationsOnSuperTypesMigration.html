Начиная с Kotlin 1.4, сообщается о бессмысленных целях аннотаций в суперклассах.

Цели аннотаций, такие как @get:, не имеют смысла в суперклассах и запрещены.

Пример:


  interface Foo
  annotation class Ann
  class E : @field:Ann @get:Ann @set:Ann @setparam:Ann Foo

После применения быстрого исправления:


  interface Foo
  annotation class Ann
  class E : Foo

Эта проверка выполняется только если уровень языка Kotlin в проекте или модуле 1.4 или выше.
