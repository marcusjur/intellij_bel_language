Сообщает о классах, реализующих java.lang.Comparable, но не переопределяющих equals().

Если метод equals() не переопределен, его реализация может быть несовместима с реализацией compareTo(). Когда объекты такого класса добавляются в коллекции типа java.util.SortedSet, это может нарушить контракт java.util.Set, определённый методом equals().

Пример:


  class Length implements Comparable<Length> {
    private int cm = 0;
    @Override
    public int compareTo(@NotNull Length o) {
      if (cm == o.cm) return 0;
      return cm < o.cm ? -1 : 1;
    }
  }

После применения быстрого исправления:


  class Length implements Comparable<Length> {
    private int cm = 0;
    @Override
    public int compareTo(@NotNull Length o) {
      if (cm == o.cm) return 0;
      return cm < o.cm ? -1 : 1;
    }
    @Override
    public boolean equals(Object o) {
      return o instanceof Length && compareTo((Length) o) == 0;
    }
  }
