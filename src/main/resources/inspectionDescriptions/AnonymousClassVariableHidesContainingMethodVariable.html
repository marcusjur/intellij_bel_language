Сообщать о полях в анонимных классах, имена которых совпадают с локальными переменными или параметрами включающего метода или лямбда-выражения.

Такое именование может привести к ошибочному использованию поля анонимного класса в случаях, когда в включающем методе намеренно используется переменная или параметр с тем же именем.

Рекомендуется переименовать поле с помощью быстрого исправления.

Пример:


  class Test {
    public Test(String value) {
      Object foo = new Object() {
        private String value = "TEST";
        public void foo() {
          System.out.println(value); //обращение к полю, а не к параметру
        }
      };
    }
  }
