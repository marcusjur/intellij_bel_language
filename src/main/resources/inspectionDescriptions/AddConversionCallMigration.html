Отчёт покажет выражения с типом Int, которые приведут к ошибкам компиляции в Kotlin 1.9 и более поздних версиях.

Пример:


  fun takeByte(x: Byte) {}
  fun foo() {
      takeByte(1 + 1) // будет преобразовано в Int в версии 1.9
  }

После применения быстрого исправления:


  fun takeByte(x: Byte) {}
  fun foo() {
      takeByte((1 + 1).toByte()) // будет преобразовано в Int в версии 1.9
  }

Проверка доступна для уровней языка Kotlin начиная с версии 1.7.
