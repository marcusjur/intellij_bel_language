Сообщает о создании новых экземпляров объектов Long, Integer, Short или Byte с примитивными аргументами типа long, integer, short или byte.

Рекомендуется использовать статический метод valueOf(), введенный в Java 5. По умолчанию этот метод кэширует объекты со значениями в диапазоне от -128 до 127 включительно.

Пример:


  Integer i = new Integer(1);
  Long l = new Long(1L);

После применения быстрого исправления код станет:


  Integer i = Integer.valueOf(1);
  Long l = Long.valueOf(1L);

Эта проверка выполняется только если уровень языка проекта или модуля установлен на 5 или выше.

Используйте опцию Игнорировать создание числовых объектов со строковыми аргументами, чтобы игнорировать вызовы числовых конструкторов с аргументами типа String.

Используйте опцию Сообщать только о вызовах устаревших конструкторов, чтобы получать уведомления только о вызовах устаревших конструкторов. Конструкторы Long, Integer, Short и Byte помечены как устаревшие начиная с JDK 9.
