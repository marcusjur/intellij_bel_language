Сообщать о неразмеченных выражениях return внутри встроенных лямбда-выражений.

Такие выражения могут вводить в заблуждение, так как не всегда понятно, к какой области видимости относится return.

Быстрое исправление Изменить на return@... доступно для автоматического исправления кода.

Пример:


  fun test(list: List<Int>) {
      list.forEach {
          // Это выражение return возвращает из функции test
          // Его можно изменить на return@forEach, чтобы изменить область видимости
          if (it == 10) return
      }
  }

После применения быстрого исправления:


  fun test(list: List<Int>) {
      list.forEach {
          if (it == 10) return@test
      }
  }
