Сообщает об отложенной инициализации static переменной небезопасным для потоков способом.

Отложенная инициализация static переменных должна выполняться с использованием соответствующих синхронизационных конструкций для предотвращения конфликтующих инициализаций разными потоками.

Где это применимо, рекомендуется использовать быстрое исправление, которое вводит идиому класса-держателя отложенной инициализации. Эта идиома использует тот факт, что JVM гарантирует, что класс не будет инициализован до его использования.

Пример:


  class X {
    private static List<String> list;
    public List<String> getList() {
      if (list == null) {
        list = List.of("one", "two", "tree");
      }
      return list;
    }
  }

После применения быстрого исправления:


  class X {
    private static final class ListHolder {
      static final List<String> list = List.of("one", "two", "tree");
    }
    public List<String> getList() {
      return ListHolder.list;
    }
  }
