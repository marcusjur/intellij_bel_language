Сообщение о ненужном использовании схемы `grpc` или стандартных схем `http[s]` в gRPC-запросах

Примеры запросов: 


# Схему `grpc` можно опустить, так как по умолчанию предполагается небезопасное соединение
GRPC grpc://localhost/TestService/testRpc


# Схему `http` можно опустить: для небезопасных соединений предпочтительна пустая схема, а для безопасных - специальная схема `grpcs`
GRPC http://localhost/TestService/testRpc


# Схему `https` следует заменить на `grpcs`: предпочтительно использовать специальную схему `grpcs`, указывающую на необходимость защиты транспортного уровня для выполнения запроса
GRPC https://localhost/TestService/testRpc

Во избежание путаницы рекомендуется использовать специальную схему `grpcs` в gRPC-запросах, когда запрос должен использовать защищенный канал связи. В противном случае схему можно полностью опустить
