Сообщает о вызовах блокирующих методов в участках кода, которые не должны блокировать поток выполнения.

Пример (проект Reactor):


Flux.just("1").flatMap(f -> {
    Flux<String> just = loadUsersFromDatabase();
    just.toIterable(); // Ошибка: вызов блокирующего оператора в неблокирующем контексте
    return just;
  }
);

Рассмотрите возможность использования соответствующего планировщика (например, Schedulers.boundedElastic()) для выполнения блокирующего кода или поищите альтернативный неблокирующий API.


Пример (Kotlin корутины):


suspend fun exampleFun() {
    Thread.sleep(100); // Ошибка: вызов блокирующего метода в suspend-функции
}

Рассмотрите возможность использования специального диспетчера (например, Dispatchers.IO) для выполнения блокирующего кода или поищите альтернативный неблокирующий API.

Настройка проверки:

  • В списке блокирующих аннотаций укажите аннотации, помечающие блокирующие методы.
  • В списке неблокирующих аннотаций укажите аннотации, помечающие неблокирующие методы.

Указанные аннотации могут использоваться как внешние аннотации
