Сообщает о вызовах методов equals(), compareTo() или подобных методов, сравнивающих объект с самим собой. Контракты этих методов указывают, что такие вызовы всегда будут возвращать true для equals() или 0 для compareTo(). Проверка также распространяется на вызовы методов Objects.equals(), Objects.deepEquals(), Arrays.equals(), Comparator.compare(), методы assertEquals() тестовых фреймворков (JUnit, TestNG, AssertJ), Integer.compare(), Integer.compareUnsigned() и подобные методы.

Обратите внимание, что в редких случаях проверка может сообщать о вызовах equals(), возвращающих false, поскольку, хотя выражения идентичны с обеих сторон, они создают отдельные объекты, и сравнение выполняется по ссылкам, а не по содержимому. Простейший пример: new Object().equals(new Object()). В любом случае, такие вызовы подозрительны и могут иметь другое назначение.

Пример:


  class Foo {
    boolean foo(Object o) {
        return o.equals(o); // Предупреждение
    }
    boolean bar(String[] ss) {
        return Arrays.equals(ss, ss); // Предупреждение
    }
}
