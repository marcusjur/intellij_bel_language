Сообщает о выражениях, затронутых распаковкой (автоматическое преобразование объекта в примитивное значение). Рекомендуется использовать примитивные типы вместо объектов. Это может существенно повлиять на производительность.

Пример:

int x = new Integer(42);

Быстрое исправление сделает преобразование явным:

int x = new Integer(42).intValue();

Автораспаковка появилась в Java 5. Эта проверка помогает обеспечить обратную совместимость с более ранними версиями Java.
