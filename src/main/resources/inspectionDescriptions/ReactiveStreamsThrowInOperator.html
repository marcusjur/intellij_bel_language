Сообщение о выражениях throw в коде операторов Reactor/RxJava.

Выброс исключений из операторов Reactor/RxJava может указывать на проблему, поскольку вы можете вернуть "реактивноподобную" ошибку: Mono.error() или Flowable.error() из flatMap(), или вызвать sink.error() из оператора handle() в Reactor.

Кроме того, фабричные методы Reactor позволяют возвращать проверяемые исключения без ошибок, в то время как выброс таких исключений без класса Exceptions приведет к ошибке компиляции.

Пример:


Flux.just(1, 2, 3).flatMap(i -> {
            throw new RuntimeException();
        })

После применения быстрого исправления:


Flux.just(1, 2, 3).flatMap(i -> {
            return Flux.error(new RuntimeException());
        })

Изменения в версии 2019.3
