Сообщает о вызовах метода super.tearDown() в JUnit 3, которые не выполняются внутри блока finally. Если исключение возникает до вызова super.tearDown(), это может привести к несогласованности и утечкам.

Пример:


  public class AnotherTest extends CompanyTestCase {
    private Path path;
    @Override
    protected void setUp() throws Exception {
      super.setUp();
      path = Files.createTempFile("File", ".tmp");
    }
    @Override
    protected void tearDown() throws Exception {
      Files.delete(path);
      super.tearDown();
    }
  }

Улучшенный код:


  public class AnotherTest extends CompanyTestCase {
    private Path path;
    @Override
    protected void setUp() throws Exception {
      super.setUp();
      path = Files.createTempFile("File", ".tmp");
    }
    @Override
    protected void tearDown() throws Exception {
      try {
        Files.delete(path);
      } finally {
        super.tearDown();
      }
    }
  }
