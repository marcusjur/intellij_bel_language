Обнаружена ошибка передачи блокировки по значению.

Случайное копирование значения, содержащего блокировку, может привести к некорректной работе обеих копий. Обычно такие значения должны передаваться по ссылке через указатель. Здесь под блокировкой подразумеваются типы, реализующие интерфейс sync.Locker, например sync.Mutex или sync.WaitGroup.

Пример:

type SafeInt struct {
  m sync.Mutex
  i int
}
func (s SafeInt) Inc() { // мьютекс копируется
  s.m.Lock()
  s.i++
  s.m.Unlock()
}

После применения быстрого исправления Добавить указатель:

type SafeInt struct {
  m sync.Mutex
  i int
}
func (s *SafeInt) Inc() {
  s.m.Lock()
  s.i++
  s.m.Unlock()
}
