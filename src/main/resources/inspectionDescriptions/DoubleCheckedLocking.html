Сообщение о двойной проверке блокировки.

Двойная проверка блокировки пытается инициализовать поле по требованию потокобезопасным способом, избегая при этом накладных расходов на синхронизацию. К сожалению, она не является потокобезопасной при использовании с полями, не объявленными как volatile. В Java 1.4 и более ранних версиях двойная проверка блокировки не работает даже с полями volatile. Прочтите статью по ссылке выше для подробного объяснения проблемы.

Пример неправильной двойной проверки блокировки:


  class Foo {
    private Helper helper = null;
    public Helper getHelper() {
      if (helper == null)
        synchronized(this) {
          if (helper == null) helper = new Helper();
        }
        return helper;
      }
    }
    // Другие функции и члены класса...
  }
