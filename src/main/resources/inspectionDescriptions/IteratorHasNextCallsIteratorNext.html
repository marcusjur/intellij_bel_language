Сообщает о реализациях Iterator.hasNext() или ListIterator.hasPrevious(), которые вызывают Iterator.next() или ListIterator.previous() на экземпляре итератора. Такие вызовы почти наверняка являются ошибкой, поскольку методы типа hasNext() не должны изменять состояние итератора, тогда как next() должен это делать.

Пример:


  class MyIterator implements Iterator<Integer> {
    public boolean hasNext() {
      return next() != null;
    }
  }
