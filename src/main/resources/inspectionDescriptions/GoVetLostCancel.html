Сообщает о путях выполнения, где не вызывается функция cancel, возвращаемая функциями context.WithCancel и подобными.

Функции WithCancel, WithDeadline и WithTimeout принимают Context (родительский) и возвращают производный Context (дочерний) и CancelFunc. Вызов CancelFunc отменяет дочерний контекст и его потомков, удаляет ссылку родителя на дочерний элемент и останавливает все связанные таймеры. Если не вызвать CancelFunc, это приведет к утечке дочернего контекста и его потомков до тех пор, пока родительский контекст не будет отменен или не сработает таймер.

Подробную информацию о пакете context смотрите в разделе Package context на go.dev.

Пример:

func _(ctx context.Context, cancel func()) {
 var ctx2 context.Context
 ctx2, cancel = context.WithCancel(ctx)
 _ = ctx2
}
