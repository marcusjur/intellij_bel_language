Сообщает о ссылках на нестатические private члены из вложенного класса на внешний класс. Для таких ссылок javac генерирует синтетические методы доступа с уровнем package-private, что может создать угрозу безопасности, поскольку члены, которые кажутся private, фактически доступны во всем пакете.

Вложенные классы и их внешние классы компилируются в отдельные файлы классов. Java Virtual Machine обычно запрещает доступ к private полям и методам одного класса из другого класса. Чтобы обеспечить доступ к private членам внешнего класса из вложенного класса, javac создает синтетический метод доступа уровня package-private.

Изменение уровня доступа private членов на package-private делает фактическую доступность явной. Это также экономит небольшое количество памяти и может улучшить производительность в условиях ограниченных ресурсов.

Эта проверка выполняется только если уровень языка проекта или модуля равен 10 или ниже. В Java 11 и выше методы доступа больше не генерируются благодаря вложенному контролю доступа (JEP 181).

Пример:


  class Outer {
    private void x() {}
    class Inner {
      void y() {
        x();
      }
    }
  }

После применения быстрого исправления:


  class Outer {
    void x() {}
    class Inner {
      void y() {
        x();
      }
    }
  }
