Обнаружено использование устаревших названий диагностик в аннотации, например @Suppress("HEADER_WITHOUT_IMPLEMENTATION").

Некоторые диагностики из Kotlin 1.2 и более ранних версий устарели, что делает такие подавления избыточными.

Пример:


@Suppress("HEADER_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}

После применения быстрого исправления:


@Suppress("EXPECTED_DECLARATION_WITH_BODY")
expect fun connection() {
  // ...
}
