Сообщает о вызовах, которые имеют смысл только для null-допустимых получателей, при их использовании на non-null получателях.

Некоторые функции стандартной библиотеки, такие как orEmpty() или isNullOrEmpty, имеют смысл только при вызове на null-допустимых типах. В остальных случаях их можно опустить, так как результат будет одинаковым.

Удалить избыточный вызов и Изменить вызов на ... доступны как быстрые исправления для автоматического исправления кода.

Пример:


  fun test(s: String) {
      val x = s.orEmpty() // быстрое исправление упростит до 's'
      val y = s.isNullOrEmpty() // быстрое исправление упростит до 's.isEmpty()'
  }
