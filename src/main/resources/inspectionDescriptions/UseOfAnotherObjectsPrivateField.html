Сообщает о доступе к полям private или protected другого объекта. Java разрешает доступ к таким полям объектов того же класса, что и текущий объект, однако некоторые стили кодования не рекомендуют такое использование. Кроме того, такой прямой доступ к полям private может не работать в компонентно-ориентированных архитектурах (например, Spring или Hibernate), которые ожидают, что весь доступ к другим объектам осуществляется через вызовы методов, чтобы фреймворк мог использовать прокси для посредничества доступа.

Пример:


  public class Base {
    protected int bar;
    void increment(Base base) {
      bar++;
      base.bar++; // warning: direct access to another object's non-public field
    }
  }
Поле может быть инкапсулировано с помощью быстрого исправления.

Настройка проверки:

  • Используйте опцию Игнорировать доступ из того же класса, чтобы игнорировать доступ из того же класса и сообщать только о доступе из внутренних или внешних классов.

    Чтобы игнорировать доступ из внутренних классов, используйте вложенную опцию Игнорировать доступ из внутренних классов.

  • Используйте Игнорировать доступ из метода 'equals()', чтобы игнорировать доступ из метода equals().
