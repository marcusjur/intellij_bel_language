Сообщать о вызовах java.lang.Runtime.exec(), в которых динамически сконструированная строка используется как выполняемая команда.

Динамически сконструированные строки для выполнения команд являются распространенным источником уязвимостей безопасности. По умолчанию эта проверка игнорирует константы времени компиляции.

Пример:


  String i = getUserInput();
  Runtime runtime = Runtime.getRuntime();
  runtime.exec("foo" + i); // выдаст предупреждение

В настройках проверки можно указать, чтобы любые поля static final рассматривались как константы. Будьте осторожны, так как при включении этой опции такие строки будут игнорироваться: 


  static final String COMMAND = "ping " + getDomainFromUserInput() + "'";
