Класс может быть преобразован в record-класс.

Record-классы ориентированы на моделирование неизменяемых данных, а не расширяемого поведения. Автоматическая неявная реализация методов, управляемых данными (например, equals() и аксессоры), помогает уменьшить шаблонный код.

Обратите внимание, что не каждый класс может быть record-классом. Существуют следующие ограничения:

  • Класс не может иметь подклассы.
  • Все нестатические поля класса должны быть final.
  • Не должно быть инициализаторов, обобщенных конструкторов и нативных методов.

Полное описание record-классов см. в спецификации языка Java.

Пример:


  class Point {
    private final double x;
    private final double y;
    Point(double x, double y) {
      this.x = x;
      this.y = y;
    }
    double getX() {
      return x;
    }
    double getY() {
      return y;
    }
  }

После применения быстрого исправления:


  record Point(int x, int y) {
  }

Включение опции Предложить переименование методов доступа автоматически переименует аксессоры getX()/isX() в x().

Используйте опцию Если члены становятся более доступными, чтобы указать действия при увеличении доступности членов класса:

  • Выберите Не предлагать преобразование, чтобы не выполнять преобразование, когда члены становятся более доступными.
  • Выберите Показать конфликты, чтобы отобразить затронутые члены и запросить подтверждение. В пакетном режиме преобразование не будет предложено.
  • Выберите Преобразовать без уведомления, чтобы при необходимости повысить доступность без уведомлений.

Используйте список Запретить преобразование классов со следующими аннотациями, чтобы исключить классы, аннотации которых соответствуют указанным шаблонам.
