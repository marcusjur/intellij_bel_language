Сообщает о методах, имена которых слишком короткие, слишком длинные или не соответствуют указанному шаблону регулярного выражения.

Эта проверка игнорирует методы экземпляра, которые переопределяют библиотечные методы и конструкторы.

Пример: Если проверка включена для статических методов, и минимальная длина имени метода установлена на 4 (значение по умолчанию), следующий статический метод вызовет предупреждение, так как его имя имеет длину 3, что меньше 4: public static int max(int a, int b).

Быстрое исправление для переименования таких методов доступно только в редакторе.

Настройка проверки:

Используйте список в разделе Параметры, чтобы указать, какие методы следует проверять. Снимите флажки с типов методов, которые нужно пропустить при проверке. Укажите значение 0 в поле длины, чтобы пропустить соответствующую проверку.

Регулярные выражения должны быть указаны в стандартном формате java.util.regex.
