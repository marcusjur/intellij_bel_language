Сообщать об операторах throw внутри блока finally. Хотя иногда это делается намеренно, такие операторы throw могут скрывать исключения, выброшенные из блоков try и catch, и усложнять процесс отладки.

Пример:


  class Fascinating {
    void doTask(Task t) {
      try {
        t.activity();
      } finally {
        if (!t.finish()) {
          throw new IllegalStateException();
        }
      }
    }
  }

Используйте опцию Предупреждать о каждом возможном месте, где может быть выброшено объявленное исключение, чтобы также получать предупреждения при вызове методов, объявляющих исключения.
