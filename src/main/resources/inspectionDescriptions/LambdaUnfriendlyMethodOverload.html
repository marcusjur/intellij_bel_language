Обнаружены перегруженные методы с конфликтующими сигнатурами абстрактных методов в используемых функциональных интерфейсах.

Такая перегрузка создает неоднозначность и требует явного приведения лямбда-выражения к определённому типу или явного указания типов параметров лямбда-выражения. Рекомендуется использовать разные имена для перегруженных методов, чтобы устранить неоднозначность.

Пример:


  interface MyExecutor {
    void execute(Supplier<?> supplier);
    void execute(Callable<?> callable);
  }

Здесь Supplier и Callable являются функциональными интерфейсами, чьи единственные абстрактные методы не принимают параметров и возвращают значение не void. Следовательно, без явного приведения типа невозможно определить тип лямбда-выражения в точке вызова.
