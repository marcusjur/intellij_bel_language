Сообщает о случаях регистрации Runnable, который захватывает ссылку, передаваемую в Cleaner.register(). Если ссылка захвачена, она никогда не станет фантомно достижимой, и операция очистки никогда не будет вызвана.

Возможные источники проблемы:

  • лямбда-выражения, использующие нестатические методы, поля или само ключевое слово this
  • нестатические внутренние классы (анонимные или нет) в Java 18 и ниже всегда захватывают эту ссылку
  • ссылки на методы экземпляра
  • доступ к нестатическим членам внешнего класса из нестатического внутреннего класса

Пример кода, который будет отмечен: 


  int fileDescriptor;
  Cleaner.Cleanable cleanable = Cleaner.create().register(this, () -> {
    System.out.println("adsad");
    //this захвачен через fileDescriptor
    fileDescriptor = 0;
  });

Эта проверка выполняется только если уровень языка проекта или модуля установлен на 9 или выше.
