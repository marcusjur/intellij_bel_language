Сообщает о вызовах конструктора BigDecimal, принимающего значения типа double. Эти конструкторы создают объект BigDecimal, который точно равен предоставленному значению double. Однако, поскольку double кодируется в формате IEEE 754 с двойной точностью (64-битное двоичное представление с плавающей точкой), точное значение может быть неожиданным.

Например, new BigDecimal(0.1) создаст объект BigDecimal. Его значение будет 0.1000000000000000055511151231257827021181583404541015625, что является ближайшим к 0.1 числом, которое может быть представлено как double. Чтобы получить объект BigDecimal со значением, идентичным тому, что написано в исходном коде, используйте new BigDecimal("0.1") или BigDecimal.valueOf(0.1).

Пример:


  class Constructor {
    void foo() {
      new BigDecimal(0.1);
    }
  }

После применения быстрого исправления:


  class Constructor {
    void foo() {
      new BigDecimal("0.1");
    }
  }
