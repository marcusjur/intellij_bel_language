Сообщение о нечётком тестовании при использовании Go SDK версии ниже 1.18

Нечёткое тестование (fuzzing) — это метод автоматизованного тестования, включающий направленный поиск входных данных, которые могут вызвать сбой программы или выявить некорректное поведение. Go поддерживает нечёткое тестование начиная с версии 1.18.

Пример нечёткого тестования:


func Div(a, b int) int {
	return a / b
}
func FuzzDiv(f *testing.F) {
	f.Fuzz(func(t *testing.T, a, b int) {
		Div(a, b) // сообщение об ошибке времени выполнения: деление на ноль
	})
}

Дополнительную информацию см. в разделе Go Fuzzing.
