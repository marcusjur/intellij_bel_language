Обнаружены все public конструкторы в классах, не являющихся public.

Обычно нет причин создавать public конструкторы в классах с более низким уровнем доступа. Однако обратите внимание, что эта проверка может изменить поведение некоторых рефлексивных вызовов. В частности, Class.getConstructor() не сможет найти обновленный конструктор (вместо этого следует использовать Class.getDeclaredConstructor()). Не используйте эту проверку, если ваш код или код используемых фреймворков зависит от доступности конструктора через getConstructor().

Пример:


  class House {
    public House() {}
  }

После применения быстрого исправления:


  class House {
    House() {}
  }
