Сообщает о проблемах в аннотации @Contract метода. Возможные типы проблем:
  • Синтаксические ошибки в контракте
  • Контракт не соответствует сигнатуре метода (неверное количество параметров)
  • Реализация метода не соответствует контракту (например, возвращает true, хотя контракт указывает false)

Пример:


  // Метод не имеет параметров, хотя контракт ожидает 1
  @Contract("_ -> fail")
  void x() {
    throw new AssertionError();
  }
