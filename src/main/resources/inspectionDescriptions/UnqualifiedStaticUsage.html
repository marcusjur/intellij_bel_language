Сообщает об использовании статических членов без указания имени класса.

Использование статических членов в том же классе является допустимым, но может вызывать путаницу.

Пример:


  class Foo {
    static void foo() {}
    static int x;
    void bar() {
      foo();
      System.out.println(x);
    }
    static void baz() { foo(); }
  }

После применения быстрого исправления:


  class Foo {
    static void foo() {}
    static int x;
    void bar() {
      Foo.foo();
      System.out.println(Foo.x);
    }
    static void baz() { Foo.foo(); }
  }

В настройках проверки можно включить или отключить отчёты для:

  • Доступа к статическим полям
    void bar() { System.out.println(x); }

  • Вызовов статических методов
    void bar() { foo(); }
    static void baz() { foo(); }

Также можно настроить проверку так, чтобы она сообщала только об использовании статических членов из нестатического контекста. В приведенном выше примере static void baz() { foo(); } не будет отмечено.
