Обнаружен код, использующий == или != вместо equals() для сравнения объектов типа Number.

Такая ошибка часто возникает при автоупаковке, когда сравниваются не примитивные типы, а экземпляры классов-оберток, например Integer вместо int.

Пример:

  void foo(Integer a, Integer b) {
    final boolean bool = a == b;
  }

Если известно, что a не является null, можно безопасно применить "небезопасное" быстрое исправление, что приведет к следующему результату:

  void foo(Integer a, Integer b) {
    final boolean bool = a.equals(b);
  }
