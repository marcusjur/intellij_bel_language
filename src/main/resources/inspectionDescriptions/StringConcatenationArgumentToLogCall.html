Сообщает о конкатенации нестатических строк, используемых в качестве аргументов методов логования SLF4J и Log4j 2. Нестатическая конкатенация строк вычисляется во время выполнения, даже если сообщение не логируется, что негативно влияет на производительность. Рекомендуется использовать параметризованные сообщения логования, которые не вычисляются, когда логование отключено.

Пример:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);
    public void saveTheWorld(int i, String s, boolean b) {
      LOG.info("saveTheWorld(" + i + ", " + s + ", "  + b + ")");
      // todo
    }
  }

После применения быстрого исправления:


  public class Vital {
    private static final Logger LOG = LoggerFactory.getLogger(Vital.class);
    public void saveTheWorld(int i, String s, boolean b) {
      LOG.info("saveTheWorld({}, {}, {})", i, s, b);
      // todo
    }
  }

Настройка проверки:

  • Используйте список уровней предупреждений для игнорирования определённых более высоких уровней логования. Более высокие уровни логования могут быть включены даже в продакшене, и аргументы всегда будут вычисляться.
