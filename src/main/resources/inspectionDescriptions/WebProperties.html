Сообщает о следующих проблемах в дескрипторах, используемых для развертывания веб-модулей на сервере:
  • Ссылки на неинстанцируемые классы
  • Ссылки на классы, не расширяющие требуемые классы
  • Ссылки на классы с неправильной областью видимости
  • Пустые теги и значения атрибутов
  • Теги и значения атрибутов, не соответствующие требуемому формату (например, идентификаторы Java)
  • Теги без обязательных дочерних тегов или атрибутов
  • Теги, определяющие объекты с повторяющимися именами

Пример:



<web-app>
  <filter-mapping>
    <filter-name>CustomFilter</filter-name> // Ошибка: должен быть определен дочерний тег servlet-name или url-pattern
  </filter-mapping>
  <servlet-mapping>
    <servlet-name>CustomServletName</servlet-name>
    <url-pattern>/ baseUrl</url-pattern> // Ошибка: URI шаблон не должен содержать пробелы
  </servlet-mapping>
</web-app>
