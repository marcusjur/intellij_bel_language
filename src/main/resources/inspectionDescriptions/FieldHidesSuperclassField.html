Сообщать о полях в производном классе, которые имеют такое же имя, как поля в суперклассе. Поля Java не могут быть переопределены в производных классах, поэтому поле в производном классе скрывает поле суперкласса.

Такое именование может привести к ошибочному использованию поля производного класса в случаях, когда предполагалось использование одноименного поля базового класса.

Рекомендуется переименовать поле в производном классе с помощью быстрого исправления.

Пример:

class Parent {
  Parent parent;
}
class Child extends Parent {
  Child parent;
}

Для этой проверки можно настроить следующие параметры:

  1. Игнорировать недоступные поля - указывает, должна ли проверка сообщать обо всех конфликтах имен или только о конфликтах с полями, видимыми в дочернем классе.
  2. Игнорировать статические поля, скрывающие статические поля - игнорировать static поля, которые скрывают static поля в базовом классе.
