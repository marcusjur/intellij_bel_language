Сообщает о следующих проблемах в конфигурационном файле `application.xml`:
  • Ссылки на неинстанцируемые классы
  • Ссылки на классы, не расширяющие требуемые классы
  • Ссылки на классы с неправильной областью видимости
  • Теги и значения атрибутов, не соответствующие требуемому шаблону (например, Java-идентификаторы)
  • Теги без обязательных дочерних тегов или атрибутов
  • Теги, определяющие объекты с повторяющимися именами

Пример:



<application>
    <module id="ExampleWebModule">
      <web> // Ошибка: должен быть определен дочерний тег "web-uri"
        <context-root>ExampleWebApp</context-root>
      </web>
    </module>
</application>
