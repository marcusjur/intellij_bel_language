Сообщает о случаях избыточного использования имени класса для обращения к статическим членам.

Квалификация через имя класса не требуется, когда статические члены доступны в текущем классе или его суперклассе. Такая квалификация может быть безопасно удалена.

Пример:


  class Foo {
    static void foo() {}
    static int x;
    void bar() {
      Foo.foo();
      System.out.println(Foo.x);
    }
    static void baz() { Foo.foo(); }
  }

После применения быстрого исправления:


  class Foo {
    static void foo() {}
    static int x;
    void bar() {
      foo();
      System.out.println(x);
    }
    static void baz() { foo(); }
  }

С помощью параметров проверки можно настроить уведомления для следующих случаев:

  • Доступ к статическим полям:
    void bar() { System.out.println(Foo.x); }

  • Вызовы статических методов:
    void bar() { Foo.foo(); }

Кроме того, можно настроить проверку так, чтобы она сообщала только об использовании статических членов в статическом контексте. В этом случае будет выдаваться предупреждение только для static void baz() { Foo.foo(); }.
