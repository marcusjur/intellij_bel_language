Сообщает о классах, которые переопределяют метод equals(), но не переопределяют метод hashCode(), или наоборот. Это может привести к проблемам при добавлении объектов класса в Collection или HashMap.

Быстрое исправление создаст стандартную реализацию отсутствующего метода.

Пример:


class StringHolder {
  String s;
  @Override public int hashCode() {
    return s != null ? s.hashCode() : 0;
  }
}

После применения быстрого исправления:


class StringHolder {
  String s;
  @Override public int hashCode() {
    return s != null ? s.hashCode() : 0;
  }
  @Override
  public boolean equals(Object o) {
    if (this == o) return true;
    if (!(o instanceof StringHolder)) return false;
    StringHolder holder = (StringHolder)o;
    if (s != null ? !s.equals(holder.s) : holder.s != null) return false;
    return true;
  }
}
