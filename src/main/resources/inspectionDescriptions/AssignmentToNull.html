Сообщает о переменных, которым присваивается значение null вне объявления.

В Java основное назначение null — представлять неинициализованные ссылочные переменные. В редких случаях явное присваивание null может помочь сборщику мусора. Однако использование null для обозначения отсутствующих, неопределённых, недопустимых значений или ненайденных элементов считается плохой практикой и может сделать ваш код более подверженным исключениям NullPointerExceptions. Вместо этого рекомендуется использовать объекты-заместители с ожидаемой семантикой или библиотечные типы вроде Optional для представления отсутствия значения.

Пример:


  Integer convert(String s) {
    Integer value;
    try {
      value = Integer.parseInt(s);
    } catch (NumberFormatException e) {
      // Предупреждение: null используется для обозначения "недопустимого значения"
      value = null;
    }
    return value;
  }

Используйте опцию игнорировать присваивание полям, чтобы игнорировать присваивание значений полям.
