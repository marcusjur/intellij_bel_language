Сообщает о попытках создания нового объекта Collection без указания начальной емкости.

Если начальная емкость не указана, используется емкость по умолчанию, что редко является оптимальным выбором. Отсутствие указания начальной емкости может привести к проблемам производительности, когда требуется перераспределение памяти и копирование данных при превышении начальной емкости. Эта проверка отслеживает создание экземпляров классов, перечисленных в настройках проверки.

Пример:


  new HashMap<String, String>();

Проверка настраивается следующими параметрами:

  • Список коллекций для проверки.
  • Игнорировать ли инициализаторы полей.
