Сообщает о незакрытых JDBC-ресурсах. Эта проверка отслеживает следующие JDBC-ресурсы: java.sql.Connection, java.sql.Statement, java.sql.PreparedStatement, java.sql.CallableStatement и java.sql.ResultSet.

По умолчанию проверка предполагает, что ресурсы могут быть закрыты любым методом, содержащим в названии 'close' или 'cleanup'.

Пример:


  ResultSet findAllElements(Connection connection) throws SQLException {
    PreparedStatement statement = connection.prepareStatement("SELECT * FROM TABLE");//оператор не закрыт
    statement.execute();
    return statement.getResultSet();
  }

Проверка настраивается следующими параметрами:

  • Разрешать ли открытие JDBC-ресурсов внутри блока try. Этот стиль менее предпочтителен, так как он более многословен, чем открытие ресурсов перед блоком try.
  • Могут ли ресурсы быть закрыты любым вызовом метода, которому ресурс передается в качестве аргумента.
