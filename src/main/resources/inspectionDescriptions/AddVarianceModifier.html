Параметры типа в отчёте могут иметь вариативность in или out.

Использование вариативности in и out обеспечивает более точный вывод типов в Kotlin и делает семантику кода более понятной.

Пример:


  class Box<T>(val obj: T)
  fun consumeString(box: Box<String>) {}
  fun consumeCharSequence(box: Box<CharSequence>) {}
  fun usage(box: Box<String>) {
      consumeString(box)
      consumeCharSequence(box) // ошибка компиляции
  }

Быстрое исправление добавит соответствующий модификатор вариативности:


  class Box<out T>(val obj: T)
  fun consumeString(box: Box<String>) {}
  fun consumeCharSequence(box: Box<CharSequence>) {}
  fun usage(box: Box<String>) ++{
      consumeString(box)
      consumeCharSequence(box) // теперь работает
  }
