Сообщает о проблемах в XML-контексте Spring Security:

  • Неразрешённые ссылки на Bean
  • Отсутствующие обязательные теги или атрибуты
  • Неверные типы свойств
  • Несоответствие свойств типа enum
  • Неверный тип ссылочного Bean

Также сообщает о следующих проблемах:

  • Ошибки конфигурации <https>
    • <http> с шаблоном '/**' должен быть последним
    • Несколько элементов <http> без атрибута 'pattern' не допускаются
  • Ошибки конфигурации <http>
    • Тег должен быть пустым, если security='none'
  • <user-service>
    • Должен иметь атрибут 'properties' или список элементов <user>
  • <ldap-server>
    • При использовании 'manager-dn' требуется 'manager-password'
    • Должен иметь 'user-context-mapper-ref' или 'user-details-class'
  • <authentication-provider>
    • Разрешен только один сервис пользователей
  • <global-method-security>
    • Значение 'pre-post-annotations' должно быть 'enabled' для использования <pre-post-annotation-handling>
    • Нельзя использовать 'mode' со значением 'aspectj' для <protect-pointcut> и <intercept-url>
    • Атрибут 'requires-channel' разрешен только для родительского <http>
  • <protect>
    • Не рекомендуется смешивать <protect> с <global-method-security>
