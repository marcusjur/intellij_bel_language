Обнаружение избыточных обратных кавычек в коде.

Некоторые ключевые слова Kotlin являются допустимыми идентификаторами в Java, например: in, object, is. Если Java-библиотека использует ключевые слова Kotlin для методов, вы всё равно можете вызывать такой метод, экранируя его обратными кавычками (`), например foo.`is`(bar). Иногда такое экранирование избыточно и может быть безопасно опущено. Эта проверка обнаруживает и сообщает о таких случаях, а также предлагает быстрое исправление "Удалить избыточные обратные кавычки", позволяющее изменить выделенный код.

Пример:


  fun `is`(x: String) {}
  fun foo() {
      `is`("bar") // 'is' - ключевое слово, обратные кавычки обязательны
  }
  fun `test that smth works as designed`() {} // допустимо, сложный идентификатор улучшает читаемость
  val `a` = 1  // обратные кавычки не нужны
