Сообщает о случаях передачи небезопасных строк в качестве SQL-запросов в методы на языках Java и Kotlin. Это может быть причиной SQL-инъекций. Список методов берется из раздела Настройки - Внедрение языка для SQL, JPA QL, Hibernate QL и PostgreSQL

Безопасными объектами считаются:

  • Строковые литералы, экземпляры интерфейсов или объекты перечислений, int и его обертки, boolean и его обертки, объекты классов
  • Результаты вызовов методов, у которых приемник и аргументы безопасны
  • Приватные поля в том же файле, которым присваиваются только строковые литералы и имеющие безопасные инициализаторы
  • Финальные поля в том же файле с безопасными инициализаторами
  • Локальные переменные, которым присваиваются безопасные объекты
    • Такие поля, локальные переменные или параметры не должны передаваться в качестве аргументов методам или использоваться как квалификаторы, либо должны быть примитивами, их обертками или неизменяемыми объектами.
Статические финальные поля считаются безопасными.

Анализ выполняется только в пределах одного файла. Пример: 


    public void save(String sql) {
        JdbcTemplate jdbcTemplate = new JdbcTemplate();
        jdbcTemplate.queryForList(sql);
    }
