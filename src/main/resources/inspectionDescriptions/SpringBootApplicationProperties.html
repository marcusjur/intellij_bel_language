Сообщает о нераспознанных и устаревших ключах конфигурации, а также недопустимых значениях в файлах конфигурации .properties приложений Spring Boot, которые могут привести к ошибкам во время выполнения.

Пример:


server.port=invalid # Сообщение 'Невозможно преобразовать 'invalid' в java.lang.Integer'

Если у устаревшего ключа конфигурации есть замена, вы можете применить быстрое исправление "Использовать ключ замены".

Пример:


logging.path=${path} # Сообщение 'Устаревшее свойство конфигурации 'logging.path''

После применения быстрого исправления:


logging.file.path=${path}

Если ключ конфигурации не определен в spring-configuration-metadata.json, вы можете применить быстрое исправление "Определить ключ конфигурации", которое создаст файл META-INF/spring-additional-configuration-metadata.json и определит необходимые ключи.

Пример:


new.key=value #Сообщение 'Не удаётся разрешить свойство конфигурации 'new.key''

После применения быстрого исправления в META-INF/spring-additional-configuration-metadata.json будет добавлено следующее:


{
  "properties": [
    {
      "name": "new.key",
      "type": "java.lang.String",
      "description": "Description for new.key."
  }
] }

Эта проверка также подсвечивает ошибки в индексной нотации для списков и отображений конфигурационных ключей.

Пример:


spring.datasource.schema[]=${schema} #Сообщение 'Отсутствует значение индекса'

Опция Токены замены позволяет определить токены для заполнителей значений в файлах конфигурации. Эти токены указываются в формате beginToken*endToken. Если * отсутствует, предполагается, что начальный и конечный токены одинаковы.

Например, по умолчанию начальный и конечный токены установлены как @, что позволяет определять заполнители вида some.property=@another.property@.

Значения внутри токенов замены (@property.key@) не будут подсвечиваться.
