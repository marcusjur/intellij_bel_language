Сообщает о пустых блоках оператора synchronized.

Пустые операторы synchronized иногда используются для ожидания освобождения определённого ресурса другими потоками. Однако нет гарантии, что после завершения пустого оператора synchronized тот же ресурс не будет получен повторно. Для правильной синхронизации ресурс должен использоваться внутри блока synchronized.

Кроме того, пустые блоки synchronized могут появиться после рефакторинга при удалении избыточного кода. В этом случае сам блок synchronized является избыточным и также должен быть удален.

Пример:


  synchronized(lock) {}

Рекомендуется удалить пустой оператор synchronized с помощью быстрого исправления.

Эта проверка отключена в JSP файлах.
