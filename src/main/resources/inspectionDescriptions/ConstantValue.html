Сообщает о выражениях и условиях, которые всегда дают одинаковый результат, такие как true, false, null или ноль. Такие выражения могут быть заменены соответствующими константными значениями. Однако в большинстве случаев они указывают на наличие ошибок в коде.

Пример:

  // Всегда true
  // Причина: использован || вместо &&
  if (x > 0 || x < 10) {}
  System.out.println(str.trim());
  // Всегда false
  // Причина: разыменование переменной происходит до проверки на null
  if (str == null) {}

Поведение проверки может контролироваться различными аннотациями, например аннотациями для null-безопасности, аннотацией @Contract, аннотацией @Range и другими.

Настройка проверки:

  • Опция Не сообщать об утверждениях, статически доказанных как всегда истинные позволяет избежать сообщений об утверждениях, которые статически доказаны как всегда истинные. Это также включает условия вида if (alwaysFalseCondition) throw new IllegalArgumentException();.
  • Опция Игнорировать операторы assert определяет, как проверка обрабатывает операторы assert. По умолчанию эта опция отключена, что означает, что утверждения рассматриваются как выполняемые (режим -ea). Если опция включена, утверждения полностью игнорируются (режим -da).
  • Опция Предупреждать, когда константы хранятся в переменных позволяет отображать предупреждения при использовании переменных, значения которых известны как константы.

До версии IntelliJ ИСРA 2022.3 эта проверка была частью проверки "Константные условия и исключения". Теперь она разделена на две проверки: "Константные значения" и "Проблемы null-безопасности и потока данных".
