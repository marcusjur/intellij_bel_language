Сообщает об отсутствии проверки отмены в циклах, методах типа forEach и ContainerUtil.process().

Выполняется только внутри методов с аннотацией com.intellij.util.concurrency.annotations.RequiresReadLock.

Пример:


@RequiresReadLock
fun doSomething() {
  ...
  for (item in items) {
        ProgressManager.checkCanceled() // должно появиться в первой строке
    ...
  }
  items.forEach {
    ProgressManager.checkCanceled() // должно появиться в первой строке
    ...
  }
  ...
}

В случае, когда между вложенными циклами нет никакого содержимого:


@RequiresReadLock
fun doSomething() {
  ...
  for (item in items) {
        // между циклами нет содержимого
        for (inner in item.inners) {
            ProgressManager.checkCanceled() // должно появиться только в первой строке внутреннего цикла
      ...
    }
  }
  ...
}

В блокирующем контексте следует использовать com.intellij.openapi.progress.ProgressManager.checkCanceled(), а в контексте корутин - com.intellij.openapi.progress.CoroutinesKt.checkCancelled().

Для получения дополнительной информации см. раздел Отмена фоновых процессов в документации SDK платформы IntelliJ.
