Сообщает о методах, в которых количество точек return превышает указанное максимальное значение. Методы со слишком большим количеством точек return могут быть запутанными и сложными для рефакторинга.

Точкой return может быть оператор return или неявный возврат в конце метода void или конструктора.

Пример:

Следующий метод будет отмечен, если разрешено только два оператора return:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user)) {
        user.setId(getId());
        return;
      } else if (cond2(user)) {
        if (cond3(user)) {
          user.setId(getId());
          return;
        }
      }
    }
  }

Рекомендуется переписать метод, чтобы сделать его более понятным:


  void doSmth(User[] users) {
    for (User user : users) {
      if (cond1(user) || cond2(user) && cond3(user)) {
        user.setId(getId());
        return;
      }
    }
  }

Настройка проверки:

  • Используйте поле Лимит точек return, чтобы указать максимальное допустимое количество точек return в методе.
  • Используйте опцию Игнорировать критические условия, чтобы пропускать критические условия. Критическое условие - это оператор if, который содержит только оператор return
  • Используйте опцию Игнорировать для метода 'equals()', чтобы пропускать точки return в методе equals().
