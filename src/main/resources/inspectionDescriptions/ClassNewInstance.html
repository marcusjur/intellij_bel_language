Обнаружен вызов метода java.lang.Class.newInstance().

Этот метод распространяет исключения, выбрасываемые конструктором без аргументов, включая проверяемые исключения. Использование этого метода позволяет эффективно обойти проверку исключений времени компиляции, выполняемую компилятором.

Рекомендуется использовать быстрое исправление для замены этого вызова на вызов метода java.lang.reflect.Constructor.newInstance(), который обрабатывает любые исключения, выбрасываемые конструктором, обертывая их в (проверяемый) java.lang.reflect.InvocationTargetException, что позволяет избежать данной проблемы.

Пример:


  clazz.newInstance()

После применения быстрого исправления:


  clazz.getConstructor().newInstance();
