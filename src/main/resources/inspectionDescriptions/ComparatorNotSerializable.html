Сообщает о классах, которые реализуют java.lang.Comparator, но не реализуют java.io.Serializable.

Если использовать несериализуемый компаратор для создания упорядоченных коллекций, таких как java.util.TreeMap или java.util.TreeSet, эти коллекции также станут несериализуемыми. Это может привести к неожиданным и трудно диагностируемым ошибкам.

Подклассы java.lang.Comparator обычно не имеют состояния, поэтому такие проблемы легко избежать, просто пометив их как сериализуемые.

Пример:


  class Foo implements Comparator { // предупреждение
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }

После применения быстрого исправления:


  class Foo implements Comparator, Serializable { // нет предупреждения
      @Override
      public int compare(Object o1, Object o2) {
          /* ... */
      }
  }
