Заменить вызов конструктора на вызов фабричного метода.

Такая замена особенно полезна, когда фабричный метод может создавать объекты любого типа, и нет необходимости включать новый экземпляр класса.

Например, когда текущий класс расширяется и должен создавать объекты подкласса. Или когда вводится кэширование и должны возвращаться существующие объекты.
